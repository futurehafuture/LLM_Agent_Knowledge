# 软链接和硬链接

## 1. 面试官到底想考什么

这题考的是文件系统基础：文件名、文件内容和 inode 之间是什么关系。面试官想看你是否知道“文件”不只是一个路径名，底层还有元数据和引用关系。

初学者可以先用“快捷方式”理解软链接，但面试时最好能讲到 inode，否则会显得停留在表面。

## 2. 零基础版解释

软链接很像桌面快捷方式。你点快捷方式，它会跳到真正的文件。如果真正的文件被删了，快捷方式还在，但点开会失效。

硬链接更像给同一个文件起了两个名字。两个名字指向同一份真实内容。删掉其中一个名字，另一个名字仍然能访问内容。只有所有名字都删掉，这份内容才真正没人能访问。

## 3. 正式概念

在类 Unix 文件系统中，文件内容和元数据由 **inode** 表示，文件名只是目录项中指向 inode 的名字。

**硬链接** 是多个目录项指向同一个 inode。它们共享同一份文件内容和元数据。删除一个硬链接，只是减少 inode 的链接计数；只有链接计数为 0 且没有进程占用时，文件数据才会被释放。

**软链接** 又叫符号链接，是一个特殊文件，文件内容保存的是目标文件路径。访问软链接时，系统会根据这个路径再去找目标文件。

| 对比项 | 硬链接 | 软链接 |
|---|---|---|
| 本质 | 多个文件名指向同一个 inode | 一个特殊文件，保存目标路径 |
| 是否有独立 inode | 通常共享目标 inode | 有自己的 inode |
| 删除原文件后 | 其他硬链接仍可访问内容 | 软链接可能失效 |
| 是否可跨文件系统 | 通常不可以 | 可以 |
| 是否可链接目录 | 通常不允许普通用户硬链接目录 | 可以链接目录 |
| 类比 | 同一份文件的多个名字 | 快捷方式 |

### 3.1 inode 是什么

inode 可以理解为文件系统给文件内容建立的“身份证”。

一个 inode 通常记录：

- 文件类型。
- 文件权限。
- 文件所有者。
- 文件大小。
- 文件时间戳。
- 数据块位置。
- 链接计数。

注意：inode 通常不直接保存文件名。文件名存在目录项中，目录项负责把“文件名”映射到 inode。

简化关系：

```text
文件名 -> inode -> 文件数据块
```

### 3.2 硬链接的底层关系

假设有一个文件 `a.txt`，inode 编号是 100。

创建硬链接：

```bash
ln a.txt b.txt
```

此时关系可以理解为：

```text
a.txt -> inode 100 -> 数据块
b.txt -> inode 100 -> 数据块
```

`a.txt` 和 `b.txt` 是两个目录项，但指向同一个 inode。修改其中任何一个，另一个看到的内容也会变，因为底层是同一份数据。

删除 `a.txt`：

```bash
rm a.txt
```

只是删除了一个目录项，inode 的链接计数减一。只要 `b.txt` 还在，数据就不会释放。

### 3.3 软链接的底层关系

创建软链接：

```bash
ln -s a.txt c.txt
```

关系可以理解为：

```text
c.txt -> 自己的 inode -> 保存路径 "a.txt"
a.txt -> 另一个 inode -> 数据块
```

软链接本身是一个独立文件，它保存的是目标路径。

如果 `a.txt` 被删除：

```text
c.txt 仍然存在，但里面保存的路径找不到目标
```

这时软链接就变成“悬空链接”。

### 3.4 为什么硬链接通常不能跨文件系统

硬链接的本质是多个目录项指向同一个 inode。而 inode 编号只在一个文件系统内部有意义。

不同文件系统各自管理自己的 inode 表。一个文件系统里的 inode 100，到了另一个文件系统里可能完全不是同一个对象。

所以硬链接通常不能跨文件系统。

软链接保存的是路径。路径可以指向另一个挂载点或文件系统，所以软链接可以跨文件系统。

### 3.5 为什么硬链接通常不能链接目录

如果允许普通用户随意给目录创建硬链接，可能破坏目录树结构，产生循环。

例如：

```text
/a/b 又硬链接回 /a
```

这样文件系统遍历时可能陷入循环，目录的父子关系也会变得混乱。

所以系统通常限制普通用户对目录创建硬链接。

### 3.6 常用命令

```bash
# 创建硬链接
ln source.txt hard_link.txt

# 创建软链接
ln -s source.txt soft_link.txt

# 查看 inode
ls -li
```

如果两个文件 inode 编号相同，通常说明它们是硬链接关系。

## 4. 面试回答模板

软链接和硬链接都是让一个文件可以通过不同路径访问，但底层机制不同。

硬链接是多个文件名指向同一个 inode，也就是同一份文件内容。删除其中一个硬链接，并不会删除真正的数据，只是链接计数减一。只要还有其他硬链接存在，文件内容就还能访问。但硬链接通常不能跨文件系统，也一般不能直接链接目录。

软链接也叫符号链接，它本身是一个独立的特殊文件，保存的是目标文件的路径。访问软链接时，系统会根据这个路径去找真正的文件。所以软链接可以跨文件系统，也可以链接目录，但如果目标文件被删除或路径变化，软链接就会失效。

简单说，硬链接是同一个文件的多个名字，软链接是指向另一个路径的快捷方式。

## 6. 常见误区

- **误区 1：只说软链接是快捷方式，不会解释硬链接。** 快捷方式只是类比，正式回答要讲 inode。
- **误区 2：认为删除原文件后硬链接也失效。** 硬链接共享 inode，只要还有链接，内容就还在。
- **误区 3：认为软链接保存文件内容。** 软链接保存的是目标路径，不是目标文件的实际内容。
- **误区 4：忽略跨文件系统。** 硬链接通常不能跨文件系统，软链接可以。

## 7. 记忆口诀

**硬链同 inode，软链存路径；删源硬还在，软链会迷路。**

## 我应该背下来的最短答案

硬链接是多个文件名指向同一个 inode，相当于同一份文件的多个名字，删除一个名字不影响其他硬链接访问内容。软链接是一个独立的特殊文件，保存目标路径，类似快捷方式，可以跨文件系统，但目标被删或路径变了就会失效。
