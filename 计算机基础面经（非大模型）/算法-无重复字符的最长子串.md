# 无重复字符的最长子串

## 1. 面试官到底想考什么

这题考的是常见算法思想：**滑动窗口**。面试官想看你能不能把“连续子串、不重复、最长”这几个条件转成代码。

如果你能秒写出来，说明你对哈希表和双指针比较熟。真正要注意的是边界：重复字符出现时，左指针不能乱跳，尤其不能往回退。

## 2. 零基础版解释

可以把字符串看成一排座位，你用一个窗口框住其中一段。

窗口右边不断往右扩，表示尝试把更多字符放进来。如果新来的字符和窗口里已有字符重复了，就要把窗口左边往右挪，直到窗口里重新没有重复字符。

每次窗口合法时，就记录当前窗口长度，最后得到最大值。

比如 `abcabcbb`：

- 先看到 `abc`，没有重复，长度是 3。
- 再看到第二个 `a`，窗口里已经有 `a`，左边要移动到第一个 `a` 后面。
- 继续维护没有重复字符的窗口。

## 3. 正式概念

题目：给定一个字符串 `s`，找出其中不含重复字符的最长连续子串长度。

核心方法：

- 使用左右指针维护一个窗口 `[left, right]`。
- 使用哈希表记录每个字符最近一次出现的位置。
- 当 `s[right]` 在窗口内重复出现时，将 `left` 移动到该字符上次出现位置的后一位。
- 每一步更新最大窗口长度。

关键条件：

```text
如果 last[s[right]] >= left
说明这个字符在当前窗口内重复了
left = last[s[right]] + 1
```

时间复杂度：**O(n)**，每个字符最多被左右指针处理常数次。

空间复杂度：**O(k)**，k 是字符集大小。

### 3.1 为什么这题适合滑动窗口

滑动窗口适合解决一类问题：

- 目标是连续区间。
- 区间需要满足某种条件。
- 右端扩张后，如果条件被破坏，可以通过移动左端恢复。

本题的条件是“窗口内没有重复字符”。右指针加入新字符后，如果发生重复，就移动左指针，直到窗口重新合法。

### 3.2 暴力解法为什么慢

暴力思路是枚举所有子串，再判断每个子串是否有重复字符。

字符串长度为 n：

- 子串数量大约是 O(n²)。
- 每个子串判断是否重复最多 O(n)。
- 总复杂度可能达到 O(n³)。

稍微优化一点，可以枚举起点，然后向右扩展并用集合判断重复，复杂度约 O(n²)。

滑动窗口的优势是：左右指针都只向右移动，总体 O(n)。

### 3.3 用集合版本怎么写

集合版本更容易理解：

```python
def lengthOfLongestSubstring(s: str) -> int:
    seen = set()
    left = 0
    ans = 0

    for right, ch in enumerate(s):
        while ch in seen:
            seen.remove(s[left])
            left += 1
        seen.add(ch)
        ans = max(ans, right - left + 1)

    return ans
```

这个版本的逻辑是：如果新字符重复，就不断移出左边字符，直到不重复。

优点是直观；缺点是遇到重复时可能一步步移动 left。

### 3.4 用哈希表记录下标的优化版

更常见的面试版本是记录字符上次出现的位置：

```python
def lengthOfLongestSubstring(s: str) -> int:
    last = {}
    left = 0
    ans = 0

    for right, ch in enumerate(s):
        if ch in last and last[ch] >= left:
            left = last[ch] + 1
        last[ch] = right
        ans = max(ans, right - left + 1)

    return ans
```

它可以让左指针一次跳到正确位置。

### 3.5 为什么要判断 `last[ch] >= left`

以 `abba` 为例：

```text
下标：0 1 2 3
字符：a b b a
```

处理到第二个 `b` 时：

- 当前窗口是 `ab`。
- 第二个 `b` 重复。
- left 从 0 移到 2。

处理最后一个 `a` 时：

- `a` 上次出现在下标 0。
- 但当前窗口是从下标 2 开始的 `ba`。
- 下标 0 的 `a` 已经不在窗口里。

所以不能把 left 移回 1。判断 `last[ch] >= left` 的作用就是确认重复字符是否还在当前窗口内。

### 3.6 如果要求返回子串本身

只需要在更新最大长度时记录起点：

```python
def longestSubstring(s: str) -> str:
    last = {}
    left = 0
    best_start = 0
    best_len = 0

    for right, ch in enumerate(s):
        if ch in last and last[ch] >= left:
            left = last[ch] + 1
        last[ch] = right

        cur_len = right - left + 1
        if cur_len > best_len:
            best_len = cur_len
            best_start = left

    return s[best_start:best_start + best_len]
```

### 3.7 子串和子序列的区别

| 概念 | 是否连续 | 例子 |
|---|---|---|
| 子串 | 必须连续 | `"abc"` 是 `"zabcd"` 的子串 |
| 子序列 | 不要求连续，但顺序不能变 | `"acd"` 是 `"abcde"` 的子序列 |

这题问的是最长子串，所以必须连续，适合滑动窗口。

## 4. 面试回答模板

这题我会用滑动窗口。因为题目要求的是连续子串，所以可以用左右指针维护一个当前没有重复字符的窗口。

我用一个哈希表记录每个字符上一次出现的位置。右指针从左到右遍历字符串，如果当前字符之前出现过，并且上次出现位置还在当前窗口内，就说明窗口里有重复字符，需要把左指针移动到上次出现位置的后一位。然后每次更新最大长度。

需要注意的是，左指针只能向右移动，不能倒退，所以更新时要判断上次出现位置是否大于等于当前 left。

代码可以这样写：

```python
def lengthOfLongestSubstring(s: str) -> int:
    last = {}
    left = 0
    ans = 0

    for right, ch in enumerate(s):
        if ch in last and last[ch] >= left:
            left = last[ch] + 1
        last[ch] = right
        ans = max(ans, right - left + 1)

    return ans
```

## 6. 常见误区

- **误区 1：把子串和子序列混淆。** 子串必须连续，子序列可以不连续。
- **误区 2：遇到重复字符就把 left 加 1。** 正确做法是跳到重复字符上次出现位置的后一位。
- **误区 3：left 往回退。** 比如 `abba`，处理第二个 `a` 时不能把 left 从 2 退回到 1。
- **误区 4：忘记每次更新字符最新位置。**

## 7. 记忆口诀

**右扩收字符，重复左跳后；每步记最长，左针不回头。**
