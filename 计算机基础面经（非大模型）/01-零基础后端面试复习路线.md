# 零基础后端面试复习路线

## 第 1 阶段：计算机网络

要掌握的问题：

- [[计算机网络-TCP 如何保证可靠性|TCP 如何保证可靠性]]
- [[计算机网络-HTTP 版本演进|HTTP 有几个版本]]

学习目标：

- 知道 **TCP 是可靠传输协议**，HTTP 通常运行在 TCP 或 QUIC 之上。
- 能说清楚 TCP 可靠性的几个关键词：**序号、确认应答、超时重传、滑动窗口、流量控制、拥塞控制、校验和**。
- 能说清楚 HTTP/1.0、HTTP/1.1、HTTP/2、HTTP/3 的核心变化。

面试回答要求：

- 不要只说“TCP 有三次握手所以可靠”。三次握手只是建立连接，可靠性主要靠传输过程中的确认、重传、排序和控制机制。
- HTTP 版本题要讲“为什么演进”：从短连接到长连接，从队头阻塞到多路复用，从 TCP 到 QUIC。

教材式补充内容：

- 先知道网络分层：应用层、传输层、网络层、链路层。
- HTTP 属于应用层，负责请求和响应的格式。
- TCP 属于传输层，负责可靠传输。
- TCP 的可靠性要从丢包、乱序、重复、损坏、接收方处理不过来、网络拥塞这几个问题理解。
- HTTP 版本演进要从性能问题理解：建连成本、连接复用、队头阻塞、多路复用、传输延迟。

## 第 2 阶段：数据库

要掌握的问题：

- [[数据库-关系型和非关系型数据库区别|关系型和非关系型数据库区别]]
- [[数据库-MySQL 索引基础|MySQL 索引基础]]
- [[数据库-B+树索引原理|B+ 树索引原理]]

学习目标：

- 理解关系型数据库重视 **结构化表、SQL、事务、一致性**。
- 理解非关系型数据库更强调 **灵活数据模型、横向扩展、特定场景性能**。
- 知道索引本质是帮助数据库快速定位数据的数据结构。
- 理解索引不是越多越好，因为它会占空间、拖慢写入、增加维护成本。
- 理解 MySQL InnoDB 为什么常用 B+ 树索引。

面试回答要求：

- 讲索引时一定要讲“加速查询”和“写入代价”两面。
- 讲 B+ 树时一定要提 **树高低、磁盘 IO 少、叶子节点有序链表、适合范围查询**。

教材式补充内容：

- 先理解表、行、列、主键、外键、SQL。
- 再理解事务 ACID：原子性、一致性、隔离性、持久性。
- 然后理解索引：没有索引会全表扫描，有索引可以按数据结构定位。
- 最后理解 B+ 树：数据库按页读磁盘，所以索引结构要尽量降低树高，减少 IO。
- 能画出“主键索引叶子节点保存整行数据，二级索引叶子节点保存主键值”的简图。

## 第 3 阶段：操作系统

要掌握的问题：

- [[操作系统-软链接和硬链接|软链接和硬链接]]

学习目标：

- 知道文件名、文件内容、inode 之间的关系。
- 知道硬链接是多个文件名指向同一个 inode。
- 知道软链接是一个特殊文件，里面保存另一个路径。

面试回答要求：

- 不要只说“软链接像快捷方式”。可以先用快捷方式类比，但正式回答要补上 inode 和路径。
- 必须能说出：硬链接一般不能跨文件系统、不能直接链接目录；软链接可以跨文件系统，但目标删除后会失效。

教材式补充内容：

- 文件名不是文件本体，文件名在目录项里。
- inode 才记录文件元数据和数据块位置。
- 硬链接是多个目录项指向同一个 inode。
- 软链接是一个独立文件，内容是目标路径。

## 第 4 阶段：算法题

要掌握的问题：

- [[算法-无重复字符的最长子串|无重复字符的最长子串]]

学习目标：

- 掌握 **滑动窗口**。
- 能解释为什么用哈希表记录字符上次出现位置。
- 能写出时间复杂度 O(n)、空间复杂度 O(k) 的代码。

面试回答要求：

- 先说思路，再写代码：右指针扩展窗口，遇到重复字符就移动左指针。
- 代码里要注意左指针只能向右移动，不能倒退。

教材式补充内容：

- 先能区分子串和子序列。
- 先讲暴力解法为什么慢。
- 再讲滑动窗口为什么能把复杂度降到 O(n)。
- 代码要能处理 `abba` 这种左指针不能回退的例子。

## 第 5 阶段：项目和实习表达

要掌握的问题：

- [[面试表达-实习经历和岗位动机|之前在哪里实习，主要做什么]]
- [[面试表达-实习经历和岗位动机|写代码用什么语言]]
- [[面试表达-实习经历和岗位动机|为什么面这个岗位]]
- [[面试表达-实习经历和岗位动机|未来想做什么方向]]

学习目标：

- 用“背景 -> 任务 -> 行动 -> 结果 -> 反思”的结构讲实习。
- 把自己的经历和岗位要求联系起来，而不是只说“我感兴趣”。
- 能自然说明自己虽然是转方向，但正在系统补基础，并且有可验证的学习和项目经历。

面试回答要求：

- 实习经历要有具体动词：负责、设计、开发、排查、优化、联调、上线。
- 岗位动机要说清楚三点：我了解这个岗位、我的经历能迁移、我愿意长期补基础和做工程实践。
- 未来方向不要太空。可以说“希望先做扎实后端开发，逐步深入数据库、分布式系统和工程效率”。

教材式补充内容：

- 实习经历要按“背景、任务、行动、结果、反思”讲。
- 转方向要承认基础曾经不系统，但重点放在已经做过的项目、实习和复习计划。
- 岗位动机不能只说兴趣，要说岗位理解、能力匹配和发展意愿。
- 反问要围绕岗位工作内容、团队培养方式和需要提前补的技术栈。

## 我应该背下来的最短答案

零基础复习后端面试，先学网络，再学数据库，再补操作系统，然后刷滑动窗口这类高频算法，最后打磨实习和岗位动机表达。每题都要做到：先讲人话，再讲术语，最后讲面试里的标准回答。
