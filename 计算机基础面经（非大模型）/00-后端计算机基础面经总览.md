## 3. 零基础学习者的建议学习顺序

1. 先学 [[计算机网络-TCP 如何保证可靠性|TCP 可靠性]]，因为它是网络八股里的核心题。
2. 再学 [[计算机网络-HTTP 版本演进|HTTP 版本]]，把应用层和传输层的关系串起来。
3. 学 [[数据库-关系型和非关系型数据库区别|关系型和非关系型数据库区别]]，先知道数据库怎么分类。
4. 学 [[数据库-MySQL 索引基础|MySQL 索引基础]]，理解索引为什么能加速查询、为什么会拖慢写入。
5. 学 [[数据库-B+树索引原理|B+ 树索引原理]]，这是索引题继续深挖的重点。
6. 学 [[操作系统-软链接和硬链接|软链接和硬链接]]，补上文件系统基础。
7. 练 [[算法-无重复字符的最长子串|无重复字符的最长子串]]，掌握滑动窗口模板。
8. 最后准备 [[面试表达-实习经历和岗位动机|实习经历和岗位动机]]，把基础知识和个人经历连起来。

## 5. 哪些适合背模板，哪些必须理解原理

| 题目 | 准备方式 | 原因 |
|---|---|---|
| HTTP 版本 | 背模板 + 理解差异 | 版本演进有固定脉络 |
| 关系型/非关系型数据库 | 背对比表 + 举例 | 面试官看你能否讲清楚适用场景 |
| 实习经历和岗位动机 | 背结构，不背死稿 | 要自然、有细节、能追问 |
| TCP 可靠性 | 必须理解原理 | 容易追问三次握手、重传、拥塞控制 |
| MySQL 索引 | 必须理解原理 | 会追问索引失效、联合索引、回表 |
| B+ 树 | 必须理解原理 | 只背“矮胖树”不够，要能解释磁盘 IO |
| 软链接/硬链接 | 理解概念 + 背区别 | 容易混淆 inode、路径、跨文件系统 |
| 无重复字符最长子串 | 必须会写代码 | 不是说思路就行，要能落地 |

## 6. 复习提醒

这份面经虽然看起来题目不多，但背后是一条典型后端基础链路：

**网络通信 -> HTTP 请求 -> 后端服务 -> 数据库查询 -> 文件系统基础 -> 算法编码 -> 项目表达**

复习时不要把每题当孤立八股。最好的状态是：面试官问 TCP，你能讲网络可靠性；问索引，你能讲查询效率和写入代价；问实习，你能把自己做过的东西和这些基础联系起来。

## 7. 教科书式复习地图

下面是这份面经背后的知识体系，不只是题目列表。

### 7.1 计算机网络主线

先理解分层：

```text
应用层：HTTP
传输层：TCP / UDP
网络层：IP
链路层：以太网 / Wi-Fi
```

这份面经只问了 HTTP 和 TCP，但背后逻辑是：

- HTTP 负责规定客户端和服务端如何表达请求与响应。
- TCP 负责在两台机器之间提供可靠传输。
- IP 负责把包从源主机送到目标主机。

面试中可以把它们串起来：

```text
浏览器访问一个接口时，应用层使用 HTTP 表达请求，传输层常用 TCP 保证可靠传输，网络层通过 IP 寻址和路由把数据送到目标机器。
```

### 7.2 数据库主线

数据库题的主线是：

```text
数据如何组织 -> 如何查询 -> 如何加速 -> 加速有什么代价 -> 底层数据结构是什么
```

对应到题目：

- [[数据库-关系型和非关系型数据库区别|关系型和非关系型数据库区别]]：数据如何组织。
- [[数据库-MySQL 索引基础|MySQL 索引基础]]：如何加速查询。
- [[数据库-B+树索引原理|B+ 树索引原理]]：索引底层为什么这样设计。

数据库面试回答要尽量同时说“收益”和“代价”。比如索引能加快查询，但会占空间、拖慢写入、增加维护成本。

### 7.3 操作系统主线

软链接和硬链接属于文件系统基础。

背后的核心关系是：

```text
路径名 -> 目录项 -> inode -> 数据块
```

只要理解 inode，就能解释：

- 为什么硬链接删除一个名字后文件还在。
- 为什么软链接目标删除后会失效。
- 为什么硬链接通常不能跨文件系统。

### 7.4 算法主线

无重复字符最长子串属于滑动窗口。

滑动窗口题的统一模式：

```text
右指针扩张窗口
条件不满足时移动左指针
每次窗口合法时更新答案
```

这类题要从暴力解法过渡到优化解法。面试官通常更喜欢听到：为什么暴力慢，滑动窗口为什么能做到 O(n)。

### 7.5 表达题主线

表达题不是背简历，而是证明匹配度。

核心结构：

```text
我做过什么 -> 我学到了什么 -> 和岗位有什么关系 -> 我下一步怎么成长
```

转方向时尤其要注意：不要只强调兴趣，要强调行动和证据。
