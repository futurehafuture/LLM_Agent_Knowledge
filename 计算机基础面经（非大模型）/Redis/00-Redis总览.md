# Redis 面试知识体系总览

## 1. Redis 是什么，为什么面试必考

Redis（Remote Dictionary Server）是一个开源的、基于内存的键值存储系统，支持丰富的数据结构。它由意大利开发者 Salvatore Sanfilippo 在 2009 年创建，现在是后端开发中最常用的缓存和消息队列中间件之一。

后端面试中，Redis 几乎是必考内容。原因很简单：

- 互联网系统普遍用 Redis 做缓存，面试官需要判断你是否会设计缓存层。
- Redis 涉及数据结构、并发、持久化、分布式等多个核心领域，一道题可以追问很深。
- Redis 的实现原理（单线程、IO 多路复用、跳表等）是经典的设计问题，能区分基础选手和深度选手。

## 3. 高频面试题汇总

**必须能流畅回答：**

- Redis 为什么这么快？
- Redis 的数据类型有哪些，分别适合什么场景？
- Redis 如何实现持久化？RDB 和 AOF 的区别？
- 缓存穿透、缓存击穿、缓存雪崩是什么，怎么解决？
- Redis 内存不够用了怎么办？
- Redis 如何实现分布式锁？
- Redis 主从复制的原理是什么？

**中频但能拉开差距：**

- Redis 的过期键是怎么删除的？
- Redis 的 Sorted Set 底层是什么数据结构？
- Redis Cluster 数据分片原理是什么？
- Redis 事务和数据库事务有什么区别？
- Redis 单线程为什么还这么快？

## 4. Redis 和 MySQL 的定位区别

这是很多面试会问的背景问题：Redis 和 MySQL 怎么配合使用？

```text
MySQL：持久化存储，保证数据不丢失，支持复杂查询和事务
Redis：内存缓存，响应极快，适合高频读操作
```

典型架构：

```text
用户请求
   ↓
先查 Redis 缓存
   ↓ 命中 → 直接返回
   ↓ 未命中
查询 MySQL
   ↓
写入 Redis 缓存
   ↓
返回用户
```

## 我应该背下来的最短答案

Redis 是基于内存的键值存储，核心优势是速度极快（纯内存操作 + 单线程 + IO 多路复用）。它支持 String、List、Hash、Set、Sorted Set 五种核心数据类型，常用于缓存、计数器、排行榜、消息队列等场景。面试重点包括数据类型使用场景、缓存三大问题（穿透/击穿/雪崩）的解决方案、持久化机制（RDB/AOF）以及高可用方案（哨兵/集群）。
