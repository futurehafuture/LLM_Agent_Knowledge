## 先讲人话版

题目问的是：

> 在一棵二叉树里，找有多少条“向下走”的路径，路径和等于 targetSum。

注意路径不一定从根节点开始，也不一定到叶子节点结束，但必须从父节点往子节点方向走。

暴力想法：

> 以每个节点为起点，往下找所有路径和。

这样最坏可能是 $`O(n^2)`$。

优化想法：

> DFS 从根往当前节点走时，维护“根到当前节点的前缀和”。如果当前前缀和是 `cur`，以前某个祖先前缀和是 `cur - target`，那么这两个点中间的路径和就是 target。

## 必备前置知识

| 概念 | 短定义 | 为什么重要 |
|---|---|---|
| DFS | 深度优先遍历树 | 需要沿根到当前节点维护状态 |
| 前缀和 | 从路径起点累加到当前位置的和 | 两个前缀和之差表示一段路径和 |
| 回溯 | 递归返回时撤销本层影响 | 哈希表只能代表当前路径，不能混入其他分支 |
| 哈希表计数 | 记录某个前缀和出现几次 | 同一个前缀和可能在当前路径出现多次 |

## 核心原理

### 1. 暴力解法

每个节点都作为起点，向下 DFS：

```text
for each node:
    count_paths_starting_from(node, target)
```

问题：

- 平衡树可能接近 $`O(n \log n)`$。
- 链状树会退化成 $`O(n^2)`$。

面试可以先提暴力，再说优化：

> 暴力会重复计算大量从祖先到后代的路径和，所以我用前缀和把“任意起点到当前节点”的路径查询变成哈希表查找。

### 2. 前缀和转化

假设从根到当前节点的前缀和是：

```text
cur = root -> ... -> current 的路径和
```

如果某个祖先之前的前缀和是：

```text
prev = cur - target
```

那么：

```text
cur - prev = target
```

也就是从“那个祖先的下一个节点”到当前节点的路径和等于 target。

所以每到一个节点，只要查：

```python
ans += prefix_count[cur - target]
```

### 3. 为什么初始 `prefix_count[0] = 1`

如果从根到当前节点本身就等于 target：

```text
cur = target
cur - target = 0
```

这时需要有一个“空前缀和 0”来匹配，表示路径从根节点开始。因此初始化：

```python
prefix_count = {0: 1}
```

### 4. 为什么要回溯

DFS 走完左子树再走右子树时，左子树里的前缀和不能影响右子树。

错误例子：

```text
    root
   /    \
 left  right
```

`left` 分支的前缀和并不是 `right` 分支的祖先。如果不撤销，会把不同分支拼成一条不存在的路径。

所以递归流程必须是：

```text
进入当前节点：
    cur += node.val
    查询 cur - target
    prefix_count[cur] += 1

递归左右子树

离开当前节点：
    prefix_count[cur] -= 1
```

## 面试怎么答

### 30 秒版

> 这题路径可以从任意节点开始，但必须向下。暴力是每个节点作为起点再 DFS，最坏 $`O(n^2)`$。优化用前缀和：DFS 时维护根到当前节点的路径和 `cur`，如果之前出现过 `cur - targetSum`，说明从那个祖先后面到当前节点的路径和等于 target。用哈希表记录当前递归路径上的前缀和次数，进入节点加一，离开节点减一，防止不同分支互相污染。时间 $`O(n)`$，空间 $`O(h)`$ 到 $`O(n)`$。

### 2 分钟版

> 我先想到暴力：对每个节点都向下找所有路径，这样链状树会退化到 $`O(n^2)`$。进一步观察，任意一条从祖先到当前节点的路径和，其实可以表示成两个前缀和之差。假设当前从根到节点的前缀和是 `cur`，如果当前路径上某个祖先之前的前缀和是 `cur - targetSum`，那么中间这段路径和就是 target。
>
> 所以我 DFS 遍历树，维护一个 `prefix_count`，表示当前根到节点路径上每个前缀和出现了几次。每到一个节点先更新 `cur`，答案加上 `prefix_count[cur - targetSum]`，然后把 `cur` 放进哈希表，递归左右子树。递归结束后要把 `cur` 计数减掉，因为回到父节点后，这个前缀和不属于其他兄弟分支。

## 代码

### Python 标准写法

```python
from collections import defaultdict
from typing import Optional


class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def pathSum(self, root: Optional[TreeNode], targetSum: int) -> int:
        prefix_count = defaultdict(int)
        prefix_count[0] = 1
        ans = 0

        def dfs(node: Optional[TreeNode], cur_sum: int) -> None:
            nonlocal ans
            if node is None:
                return

            cur_sum += node.val

            ans += prefix_count[cur_sum - targetSum]

            prefix_count[cur_sum] += 1
            dfs(node.left, cur_sum)
            dfs(node.right, cur_sum)
            prefix_count[cur_sum] -= 1

        dfs(root, 0)
        return ans
```

### 为什么查询要在加入当前前缀和之前

如果 `targetSum = 0`，并且你先执行：

```python
prefix_count[cur_sum] += 1
ans += prefix_count[cur_sum - targetSum]
```

那么当前节点的前缀和会匹配到自己，等价于统计了一条长度为 0 的路径，这是错的。

所以顺序必须是：

1. 更新当前前缀和。
2. 查询历史前缀和。
3. 加入当前前缀和。
4. 递归子树。
5. 撤销当前前缀和。

## 复杂度

| 项目 | 复杂度 | 解释 |
|---|---|---|
| 时间 | $`O(n)`$ | 每个节点访问一次，哈希查询均摊 $`O(1)`$ |
| 空间 | $`O(h)`$ 到 $`O(n)`$ | 递归栈和当前路径前缀和；链状树最坏 $`O(n)`$ |

## 常见追问

### Q1：为什么不是滑动窗口？

树路径不是线性数组，而且节点值可以为负数。即使在数组里，有负数时也不能用经典双指针滑动窗口保证单调性。前缀和更通用。

### Q2：路径能不能从子节点往父节点走？

题目不允许。路径必须从父节点到子节点方向向下，但起点不一定是根。

### Q3：为什么哈希表里要存次数，而不是 set？

因为当前路径上可能多个不同位置有相同前缀和，它们都能形成不同路径。只存 set 会漏数。

### Q4：如果树很深递归爆栈怎么办？

Python 面试通常接受递归。工程里可以提高递归限制或改成显式栈，但显式栈要同时维护进入/退出状态来做回溯，代码更复杂。

## 容易踩坑

- 忘记 `prefix_count[0] = 1`，导致从根开始的路径漏掉。
- 忘记回溯 `prefix_count[cur_sum] -= 1`，导致左右子树路径串起来。
- 先加入当前前缀和再查询，target 为 0 时会统计空路径。
- 以为路径必须从根到叶子，这题不是。
- 没考虑负数，错误使用滑动窗口。
- 使用全局变量时忘记 `nonlocal ans`。

## 复习清单

- 暴力：每个节点作为起点 DFS，最坏 $`O(n^2)`$。
- 前缀和：`cur - prev = target`。
- 查询：`ans += prefix_count[cur_sum - targetSum]`。
- 初始化：`prefix_count[0] = 1`。
- 顺序：查询历史 -> 加当前 -> 递归 -> 撤销当前。
- 时间 $`O(n)`$，空间最坏 $`O(n)`$。
