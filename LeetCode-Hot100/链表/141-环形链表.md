# 141. 环形链表

> 链接：https://leetcode.cn/problems/linked-list-cycle/

## 题目描述

给你一个链表的头节点 `head`，判断链表中是否有环。如果链表中存在环，则返回 `true`；否则，返回 `false`。

> 如果链表中有某个节点，可以通过连续跟踪 `next` 指针再次到达，则链表中存在环。

### 示例

```
输入：head = [3,2,0,-4]，tail 连接到索引 1 的节点
输出：true
```

```
输入：head = [1,2]，tail 连接到索引 0 的节点
输出：true
```

```
输入：head = [1]，tail 不连接任何节点
输出：false
```

约束：节点数在 `[0, 10^4]` 范围内，`pos` 为 `-1` 或链表中的有效索引。

**进阶**：用 O(1) 内存解决？

---

## 思路推导

### 方法一：哈希表（O(n) 空间）

遍历每个节点，把访问过的节点存入集合。如果遇到已在集合中的节点，说明有环。

### 方法二：快慢指针（O(1) 空间，推荐）

**龟兔赛跑算法（Floyd's Cycle Detection）**：

- `slow` 每次走 1 步，`fast` 每次走 2 步。
- 若无环：`fast` 会先到达 `None`，循环结束返回 `false`。
- 若有环：`fast` 进入环后，两者都在环中运动。每轮 `slow` 走 1 步、`fast` 走 2 步，快指针每轮缩短 1 步追赶慢指针，**必然相遇**。

**为什么一定相遇？**

设环长为 L，fast 比 slow 每轮多走 1 步（相对 slow 来说），最多 L 轮就能追上，不会"错过"。

---

## Python 代码

### 写法一：哈希表

```python
class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        seen = set()
        curr = head
        while curr:
            if curr in seen:
                return True
            seen.add(curr)
            curr = curr.next
        return False
```

### 写法二：快慢指针（O(1) 空间）

```python
class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        slow = fast = head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            if slow is fast:
                return True
        return False
```

## 复杂度分析

| 写法 | 时间 | 空间 |
|---|---|---|
| 哈希表 | O(n) | O(n) |
| 快慢指针 | O(n) | O(1) |

---

## 易错点

1. **终止条件是 `fast and fast.next`**：`fast.next` 不检查就调用 `fast.next.next` 会 `AttributeError`；只检查 `fast` 不检查 `fast.next` 会在偶数长链表末尾报错。
2. **用 `is` 比较而非 `==`**：`slow == fast` 比较值，链表中可能有值相同的不同节点；`slow is fast` 比较对象引用，才是判断是否为同一节点。
3. **初始化 `slow = fast = head`**：不要一开始就让 fast 先走，先走会导致循环前相遇检查跳过了 head 的情况。

## 知识点补充

### Floyd 判圈算法

Floyd 算法实际上有两个阶段：
1. **判断是否有环**（本题）：快慢指针相遇即有环。
2. **找入环起点**（[[142-环形链表II]]）：相遇后让一个指针回 head，两者同速再走，第二次相遇即是入环口。

这个两阶段算法是面试的高频考点，两题连着看才完整。
