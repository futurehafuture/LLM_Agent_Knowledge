# 142. 环形链表 II

> 链接：https://leetcode.cn/problems/linked-list-cycle-ii/

## 题目描述

给定一个链表的头节点 `head`，返回链表开始入环的第一个节点。如果链表无环，则返回 `null`。

> 不允许修改链表。

### 示例

```
输入：head = [3,2,0,-4]，tail 连接到索引 1
输出：索引 1 处的节点（值为 2）
```

```
输入：head = [1,2]，tail 连接到索引 0
输出：索引 0 处的节点（值为 1）
```

```
输入：head = [1]，无环
输出：null
```

## 思路推导

### Floyd 判圈第二阶段：找入环口

**第一步**：快慢指针检测是否有环，找到相遇点。

**第二步**：让一个指针回到 `head`，另一个留在相遇点，两者都以步长 1 继续走，第二次相遇的节点就是入环口。

### 数学推导

设：
- `head` 到入环口的距离为 `a`
- 入环口到相遇点（沿环方向）的距离为 `b`
- 相遇点到入环口（沿环方向继续走）的距离为 `c`，则环长 `L = b + c`

快慢相遇时：
- `slow` 走了 `a + b` 步
- `fast` 走了 `a + b + k·L`（fast 在环里多绕了 k 圈，k ≥ 1）
- `fast = 2 * slow` → `a + b + kL = 2(a + b)` → `kL = a + b` → `a = kL - b = (k-1)L + (L - b) = (k-1)L + c`

**结论**：从 `head` 走 `a` 步 = 从相遇点继续走 `a` 步（绕若干整圈后抵达入环口）。

因此两指针同速走，必在入环口相遇。

```
链表结构示意：
head → ... → [入环口] → ... → [相遇点] → ... → [入环口]
              ←─── a ───→       ←── b ──→         ←─ c ─→
```

## Python 代码

### 写法一：Floyd 判圈（O(1) 空间，推荐）

```python
class Solution:
    def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
        slow = fast = head
        # 第一阶段：找相遇点
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            if slow is fast:
                break
        else:
            return None   # 无环
        # 第二阶段：找入环口
        slow = head
        while slow is not fast:
            slow = slow.next
            fast = fast.next
        return slow
```

### 写法二：哈希表（O(n) 空间）

```python
class Solution:
    def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
        seen = set()
        curr = head
        while curr:
            if curr in seen:
                return curr
            seen.add(curr)
            curr = curr.next
        return None
```

## 易错点

1. **第一阶段 while 的写法**：用 `while ... else` 结构更清晰；用 `if slow is fast: break` 代替 `return`，退出循环后统一进入第二阶段，而不是在循环内部直接 `return`。
2. **第二阶段让 `slow` 回 head，`fast` 留在相遇点**，两者都走 1 步。不能让 `fast` 也回 head，那样就是两个指针同时从 head 出发，变成了另一道题。
3. **如果 head 本身就是入环口**（`a = 0`）：数学公式依然成立，`slow` 停在 `head`，`fast` 走 `c = L` 步也回到入环口，两者在 `head` 相遇，结果正确。

## 知识点补充

### 为什么 k=1 时结论最直观？

当 fast 只比 slow 多绕 1 圈（k=1）时：
- `a = c`
- 从 head 到入环口的距离 = 相遇点沿环走到入环口的距离

记住这个 `a = c` 的直觉（k=1 场景），面试时更容易推导。
