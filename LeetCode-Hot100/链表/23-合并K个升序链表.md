# 23. 合并 K 个升序链表

> 链接：https://leetcode.cn/problems/merge-k-sorted-lists/

## 题目描述

给你一个链表数组，每个链表都已经按升序排列。请你将所有链表合并到一个升序链表中，返回合并后的链表。

### 示例

```
输入：lists = [[1,4,5],[1,3,4],[2,6]]
输出：[1,1,2,3,4,4,5,6]
```

```
输入：lists = []
输出：[]
```

```
输入：lists = [[]]
输出：[]
```

约束：`k == lists.length`，`0 <= k <= 10^4`，`0 <= lists[i].length <= 500`，总节点数不超过 `10^4`。

---

## 思路推导

### 方法一：逐一合并（暴力）

循环调用 [[21-合并两个有序链表]] 把结果合并进来。

**时间复杂度**：每次合并长度逐渐增长，总共 O(kN)（N 为所有节点数），k 大时很慢。

### 方法二：优先队列（最小堆）

维护一个大小为 k 的最小堆，每次从堆中取出当前最小的节点，接到结果链表，再把该节点的 next 压入堆。

**时间**：每个节点进出堆一次，每次 O(log k)，总共 O(N log k)。

### 方法三：分治归并

类似归并排序，两两合并：

```
第一轮：合并 (lists[0], lists[1])，(lists[2], lists[3])，...
第二轮：合并上一轮的结果 ...
...
最终：1 条有序链表
```

共 O(log k) 轮，每轮处理 O(N) 个节点，总时间 O(N log k)，空间 O(log k)（递归栈）。

## Python 代码

### 写法一：优先队列（推荐，工程直观）

```python
import heapq

class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        dummy = ListNode(0)
        tail = dummy
        # 初始化堆：(节点值, 索引, 节点)
        # 加索引是为了当 val 相同时避免比较 ListNode 对象（Python 不支持）
        heap = []
        for i, node in enumerate(lists):
            if node:
                heapq.heappush(heap, (node.val, i, node))
        while heap:
            val, i, node = heapq.heappop(heap)
            tail.next = node
            tail = tail.next
            if node.next:
                heapq.heappush(heap, (node.next.val, i, node.next))
        return dummy.next
```

### 写法二：分治归并

```python
class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        if not lists:
            return None
        return self._merge_range(lists, 0, len(lists) - 1)

    def _merge_range(self, lists, left, right):
        if left == right:
            return lists[left]
        mid = (left + right) // 2
        l1 = self._merge_range(lists, left, mid)
        l2 = self._merge_range(lists, mid + 1, right)
        return self._merge_two(l1, l2)

    def _merge_two(self, l1, l2):
        dummy = ListNode(0)
        tail = dummy
        while l1 and l2:
            if l1.val <= l2.val:
                tail.next = l1; l1 = l1.next
            else:
                tail.next = l2; l2 = l2.next
            tail = tail.next
        tail.next = l1 or l2
        return dummy.next
```

## 复杂度分析

| 方法 | 时间 | 空间 |
|---|---|---|
| 逐一合并 | O(kN) | O(1) |
| 优先队列 | O(N log k) | O(k)（堆大小） |
| 分治归并 | O(N log k) | O(log k)（递归栈） |

其中 N 为所有链表的节点总数，k 为链表数量。

---

## 易错点

1. **堆中存 `(val, i, node)` 而非 `(val, node)`**：Python 的 `heapq` 在 val 相等时会尝试比较第二个元素，`ListNode` 没有定义比较运算符，会报 `TypeError`。加入唯一索引 `i` 后，val 相同时按 i 排，永远不需要比较 ListNode。
2. **分治中 `if left == right: return lists[left]`**：不是返回 head，而是返回 `lists[left]`（整条链表的头），基础情况是"1 条链表直接返回"。
3. **堆初始化跳过空链表**：`if node:` 判断，否则堆中会有 None。

## 知识点补充

### 逐一合并的时间复杂度分析

设 k 条链表每条长度都是 n，N = kn：
- 第 1 次合并：长度 n + n = 2n
- 第 2 次合并：2n + n = 3n
- ...
- 第 k-1 次合并：(k-1)n + n = kn

总代价：n + 2n + ... + (k-1)n = O(k²n) = O(kN)，k 大时比分治慢 O(k/log k) 倍。

### 优先队列 vs 分治

- 优先队列：在线算法，可以动态加入新链表，适合流式场景。
- 分治：离线算法，需要提前知道所有链表，代码稍复杂，空间更小。

面试首选优先队列（思路清晰、代码直观）。
