# 148. 排序链表

> 链接：https://leetcode.cn/problems/sort-list/

## 题目描述

给你链表的头节点 `head`，请你将其按**升序**排列并返回排序后的链表。

**进阶**：用 O(n log n) 时间复杂度和 O(1) 空间复杂度对链表进行排序？

### 示例

```
输入：head = [4,2,1,3]
输出：[1,2,3,4]
```

```
输入：head = [-1,5,3,4,0]
输出：[-1,0,3,4,5]
```

约束：链表节点数在 `[0, 5*10^4]` 范围内，`-10^5 <= Node.val <= 10^5`。

---

## 思路推导

### 为什么用归并排序？

链表不支持 O(1) 随机访问，因此快速排序（需要随机 pivot）和堆排序（需要索引）都不适合。

归并排序的分治特性天然适合链表：
- **分**：快慢指针找中点，从中间切断。
- **治**：递归排序左右两半。
- **合**：合并两个有序链表（[[21-合并两个有序链表]]）。

### 自顶向下（递归）：O(log n) 空间

```
sort([4,2,1,3])
= merge(sort([4,2]), sort([1,3]))
= merge([2,4], [1,3])
= [1,2,3,4]
```

递归调用栈深度 O(log n)，但空间是 O(log n) 不是 O(1)。

### 自底向上（迭代）：O(1) 空间

从步长 1 开始，每轮合并相邻的两段（长度为 step），步长每轮翻倍。

```
原始：[4, 2, 1, 3, 5, 0]
step=1：[2,4] [1,3] [0,5]
step=2：[1,2,3,4] [0,5]
step=4：[0,1,2,3,4,5]
```

## Python 代码

### 写法一：自顶向下递归（推荐，思路清晰）

```python
class Solution:
    def sortList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        # 0 或 1 个节点，直接返回
        if not head or not head.next:
            return head
        # 快慢指针找中点（切断链表）
        slow, fast = head, head.next   # fast 从 head.next 出发，slow 停在左中位数
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
        mid = slow.next
        slow.next = None               # 切断成两段
        # 递归排序
        left = self.sortList(head)
        right = self.sortList(mid)
        # 合并两个有序链表
        return self.merge(left, right)

    def merge(self, l1, l2):
        dummy = ListNode(0)
        tail = dummy
        while l1 and l2:
            if l1.val <= l2.val:
                tail.next = l1; l1 = l1.next
            else:
                tail.next = l2; l2 = l2.next
            tail = tail.next
        tail.next = l1 or l2
        return dummy.next
```

### 写法二：自底向上迭代（O(1) 空间）

```python
class Solution:
    def sortList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        # 计算链表长度
        length, curr = 0, head
        while curr:
            length += 1; curr = curr.next

        dummy = ListNode(0, head)
        step = 1
        while step < length:
            prev, curr = dummy, dummy.next
            while curr:
                # 取第一段（最多 step 个节点）
                left = curr
                right = self._split(left, step)
                curr = self._split(right, step)
                # 合并并接到 prev 后
                merged_tail = self._merge_get_tail(prev, left, right)
                prev = merged_tail
            step *= 2
        return dummy.next

    def _split(self, node, k):
        """从 node 走 k 步，断开并返回后续头节点"""
        for _ in range(k - 1):
            if node:
                node = node.next
        if not node:
            return None
        rest = node.next
        node.next = None
        return rest

    def _merge_get_tail(self, prev, l1, l2):
        """合并 l1 l2，接到 prev 后，返回合并段的尾节点"""
        while l1 and l2:
            if l1.val <= l2.val:
                prev.next = l1; l1 = l1.next
            else:
                prev.next = l2; l2 = l2.next
            prev = prev.next
        prev.next = l1 or l2
        while prev.next:
            prev = prev.next
        return prev
```

## 复杂度分析

| 写法 | 时间 | 空间 |
|---|---|---|
| 递归归并 | O(n log n) | O(log n)（调用栈） |
| 迭代归并 | O(n log n) | O(1) |

---

## 易错点

1. **快慢指针起点**：`slow = head, fast = head.next`（fast 从 head.next 出发），使得偶数长链表 slow 停在**左中位数**，确保左段不空。如果 `fast = head` 则偶数时 slow 停在右中位数，左段会多一个，某些情况下长度不均匀导致无限递归。
2. **切断链表 `slow.next = None`**：不切断就递归传入的仍是整个链表，无限循环。
3. **迭代写法的 `_split` 函数走 `k-1` 步后断开**：确保第一段正好 k 个节点（不够 k 个就取剩余全部）。

## 知识点补充

### 数组 vs 链表的排序算法适用性

| 排序算法 | 数组 | 链表 |
|---|---|---|
| 快速排序 | ✅ 最常用 | ⚠️ 需要找 pivot，partition 改写复杂 |
| 归并排序 | ✅ 稳定，O(n) 空间 | ✅ 天然适合，O(1) 空间可迭代实现 |
| 堆排序 | ✅ O(1) 空间 | ❌ 需要随机访问 |
| 插入排序 | O(n²) | O(n²)，但小数据/近有序时优秀 |
