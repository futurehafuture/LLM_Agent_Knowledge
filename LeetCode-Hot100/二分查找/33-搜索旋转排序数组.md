# 33. 搜索旋转排序数组

> 链接：https://leetcode.cn/problems/search-in-rotated-sorted-array/

## 题目描述

给定一个**无重复元素**的升序数组 `nums`，在下标 `k` 处旋转后变成 `[nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]`。

在旋转后的数组中搜索 `target`，返回其下标，不存在则返回 `-1`。

要求时间复杂度 **O(log n)**。

### 示例

```
输入：nums = [4,5,6,7,0,1,2], target = 0
输出：4
```

```
输入：nums = [4,5,6,7,0,1,2], target = 3
输出：-1
```

```
输入：nums = [1], target = 0
输出：-1
```

约束：`1 <= nums.length <= 5000`

---

## 思路推导

### 核心洞察：二分后至少有一半是有序的

旋转后的数组不是完全有序的，但有一个重要性质：

> 将数组从 `mid` 分成两半后，**至少有一半是严格升序的**。

证明：被旋转的数组只有一个"断点"（最小值位置），mid 最多穿过一次断点，因此至少有一半跨越整个段而不含断点。

### 二分过程

1. 判断 **哪一半有序**（比较 `nums[l]` 和 `nums[mid]`）
2. 判断 **target 是否在有序的那一半中**
3. 如果能确定 target 在有序的一半，就去那一半；否则去另一半。

### 判断哪一半有序

- 如果 `nums[l] <= nums[mid]`：左半 `[l, mid]` 有序
- 否则：右半 `[mid, r]` 有序

---

## Python 代码

```python
class Solution:
    def search(self, nums: List[int], target: int) -> int:
        l, r = 0, len(nums) - 1
        while l <= r:
            mid = l + (r - l) // 2
            if nums[mid] == target:
                return mid

            if nums[l] <= nums[mid]:                # 左半有序
                if nums[l] <= target < nums[mid]:   # target 在左半内
                    r = mid - 1
                else:
                    l = mid + 1
            else:                                    # 右半有序
                if nums[mid] < target <= nums[r]:   # target 在右半内
                    l = mid + 1
                else:
                    r = mid - 1
        return -1
```

## 易错点

1. **`nums[l] <= nums[mid]` 必须是 ≤ 而非 <**：当 `l == mid`（两个元素时），`nums[l] == nums[mid]`，但如果用 `<` 会错误地把左半判定为无序。
2. **target 范围判断要用开闭一致**：`nums[l] <= target < nums[mid]`——左边界用 ≤（因为 target 可能等于 `nums[l]`），右边界用 <（因为 `nums[mid]` 已经判断过不等于 target）。
3. **不能直接比较 `nums[l]` 和 `target` 来决定方向**：因为数组不是全局有序的。

---

## 知识点补充

### 为什么左半有序的判断是 `nums[l] <= nums[mid]` 而不是 `nums[0] <= nums[mid]`？

在 `l` 不断移动的过程中，比较使用的基准也应该是 `l` 而非 `0`。考虑 `[4,5,6,7,0,1,2]` target=0 的场景：

- 第一轮 mid=3 (7)：`nums[0]=4 <= 7`，左半 `[4,5,6,7]` 有序，target=0 不在左半，l=4。
- 第二轮 l=4, r=6, mid=5 (1)：`nums[4]=0 <= 1`，左半 `[0,1]` 有序，target=0 在左半，r=4。
- 第三轮 mid=4 (0)：找到。

使用 `nums[l]` 可以保证在 **当前搜索区间** 上判断，与区间动态一致。

LC153（找最小值）是本体的变种：不需要关心 target，只需要不断往包含最小值的那一半走。
