# 35. 搜索插入位置

> 链接：https://leetcode.cn/problems/search-insert-position/

## 题目描述

给定一个**无重复元素**的升序数组 `nums` 和一个目标值 `target`，如果 `target` 在数组中，返回其索引；否则返回按顺序插入 `target` 后它应该在的位置。

### 示例

```
输入：nums = [1,3,5,6], target = 5
输出：2
```

```
输入：nums = [1,3,5,6], target = 2
输出：1
解释：2 应该插入在索引 1（1 和 3 之间）
```

```
输入：nums = [1,3,5,6], target = 7
输出：4
解释：7 应该插入到数组末尾
```

约束：`1 <= nums.length <= 10^4`，`-10^4 <= nums[i], target <= 10^4`

---

## 思路推导

### 这道题的本质

题目本质上就是实现 **`lower_bound`**：找到第一个 **大于等于 target** 的位置。

- 如果 `target` 存在于数组中，`lower_bound` 返回的就是 `target` 的下标。
- 如果 `target` 不存在，`lower_bound` 返回的就是它应该被插入的位置。

### 二分过程

用左闭右闭区间 `[l, r]`：

- 当 `nums[mid] >= target` 时，mid 可能是答案（或其左边还有答案），所以 `r = mid - 1`。
- 当 `nums[mid] < target` 时，mid 及其左边都不可能是答案，所以 `l = mid + 1`。
- 循环结束时 `l = r + 1`，`l` 指向的就是第一个 ≥ target 的位置。

---

## Python 代码

### 写法一：标准 lower_bound

```python
class Solution:
    def searchInsert(self, nums: List[int], target: int) -> int:
        l, r = 0, len(nums) - 1
        while l <= r:
            mid = l + (r - l) // 2
            if nums[mid] >= target:
                r = mid - 1
            else:
                l = mid + 1
        return l
```

### 写法二：用 bisect 库

```python
from bisect import bisect_left

class Solution:
    def searchInsert(self, nums: List[int], target: int) -> int:
        return bisect_left(nums, target)
```

| 写法 | 时间 | 空间 |
|---|---|---|
| 二分查找 | O(log n) | O(1) |
| bisect 库 | O(log n) | O(1) |

---

## 易错点

1. **返回 l 还是 r？** 循环结束后 `l > r`，`l` 指向第一个 ≥ target 的位置，所以要返回 `l`。
2. **r 的初始化**：如果 `r = len(nums)`（左闭右开），则 while 条件应该是 `l < r`，`r` 更新为 `mid`。区间风格必须一致。
3. **`mid = (l + r) // 2` 的溢出风险**：Python 中整数不限精度所以安全，但面试中最好养成 `l + (r - l) // 2` 的习惯。

## 知识点补充

### lower_bound 与 upper_bound

| 函数 | 含义 | Python 等价 |
|---|---|---|
| `lower_bound(x)` | 第一个 **≥ x** 的位置 | `bisect_left` |
| `upper_bound(x)` | 第一个 **> x** 的位置 | `bisect_right` |

对于本题，"插入位置"就是 `lower_bound(target)`。

### 为什么不需要单独处理 target > max(nums) 的情况？

`lower_bound` 在 target 大于所有元素时，`l` 会一直右移，最终 `l = len(nums)`，恰好是答案（插在末尾）。代码不需要特殊判断。
