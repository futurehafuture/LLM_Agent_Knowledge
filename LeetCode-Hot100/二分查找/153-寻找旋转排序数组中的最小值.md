# 153. 寻找旋转排序数组中的最小值

> 链接：https://leetcode.cn/problems/find-minimum-in-rotated-sorted-array/

## 题目描述

给定一个**无重复元素**的升序数组，在下标 `k` 处旋转一次。找出数组中的**最小元素**。

要求时间复杂度 **O(log n)**。

### 示例

```
输入：nums = [3,4,5,1,2]
输出：1
解释：原始数组 [1,2,3,4,5]，旋转 3 次后得到 [3,4,5,1,2]。
```

```
输入：nums = [4,5,6,7,0,1,2]
输出：0
```

```
输入：nums = [11,13,15,17]
输出：11
解释：旋转 4 次（等于没转）。
```

约束：`1 <= nums.length <= 5000`

---

## 思路推导

### 核心：跟右端点比较

旋转数组的最小值就是"断点"（下降点）。

二分时：
- 如果 `nums[mid] > nums[r]`：mid 在**左半段**（较大的那段），最小值一定在 mid 右边 → `l = mid + 1`。
- 如果 `nums[mid] < nums[r]`：mid 在**右半段**（较小的那段），最小值可能是 mid 或在 mid 左边 → `r = mid`。
- 如果 `nums[mid] == nums[r]`：因为有重复元素时会出现（本题没有），单独讨论。

**关键：为什么不跟 `nums[l]` 比较？**

跟 `nums[l]` 比较无法区分 `[1,2,3,4,5]`（完全有序）和 `[2,3,4,5,1]`（旋转后）——二者都是 `nums[mid] > nums[l]`，但最小值一个在左、一个在右。跟 `nums[r]` 比较则能唯一确定。

## Python 代码

### 写法一：跟右端点比较（推荐）

```python
class Solution:
    def findMin(self, nums: List[int]) -> int:
        l, r = 0, len(nums) - 1
        while l < r:                               # 注意是 < 不是 <=
            mid = l + (r - l) // 2
            if nums[mid] > nums[r]:
                l = mid + 1                        # 最小值在 mid 右边
            else:
                r = mid                            # 最小值可能是 mid 本身
        return nums[l]
```

### 写法二：左闭右闭 + 答案变量

```python
class Solution:
    def findMin(self, nums: List[int]) -> int:
        l, r = 0, len(nums) - 1
        ans = nums[0]
        while l <= r:
            mid = l + (r - l) // 2
            ans = min(ans, nums[mid])
            if nums[mid] > nums[r]:
                l = mid + 1
            else:
                r = mid - 1
        return ans
```

## 易错点

1. **`while l < r` 而非 `l <= r`**：因为此处 r 更新为 `mid`（不是 `mid - 1`），如果用 `<=` 会死循环。
2. **`r = mid` 而非 `mid - 1`**：因为 `nums[mid]` 可能就是最小值（mid 恰好在右半段），不能跳过它。
3. **跟 `nums[r]` 比较而非 `nums[l]`**：如上所述，跟 `nums[l]` 无法处理完全有序的情况。

## 知识点补充

### 旋转数组性质的直观理解

```
原始：[0, 1, 2, 4, 5, 6, 7]
      ↑最小值                  ↑最大值

旋转 3 次：[4, 5, 6, 7, 0, 1, 2]
           |--左半段--|  |--右半段--|
           (较大值)      (较小值，含最小值)
```

规律：左半段的元素都大于右半段的任意元素。因此：
- `nums[mid] > nums[r]` → mid 在左半段 → 最小值在 mid 右边
- `nums[mid] < nums[r]` → mid 在右半段 → 最小值在 mid 或其左边

### 进阶：有重复元素（LC154）

LC154 是本题的带重复元素版。当 `nums[mid] == nums[r]` 时无法判断，需要 `r -= 1` 逐步缩小范围。最坏情况退化为 O(n)。
