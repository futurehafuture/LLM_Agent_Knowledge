# 4. 寻找两个正序数组的中位数

> 链接：https://leetcode.cn/problems/median-of-two-sorted-arrays/

## 题目描述

给定两个**正序（从小到大排好序）** 数组 `nums1` 和 `nums2`，长度分别为 `m` 和 `n`。求两个数组合并后的中位数。

要求时间复杂度 **O(log(m+n))**。

### 什么是中位数？

中位数就是把一组数从小到大排好后，位于**正中间**的那个数。

- 如果总数是**奇数**：中位数 = 最中间那一个。
  - `[1, 2, 3]` → 中位数是 `2`
- 如果总数是**偶数**：中位数 = 最中间两个数的平均值。
  - `[1, 2, 3, 4]` → 中位数是 `(2+3)/2 = 2.5`

### 示例

```
输入：nums1 = [1,3], nums2 = [2]
输出：2.0
解释：合并后 [1,2,3]，总数3个（奇数），中间那个是 2。
```

```
输入：nums1 = [1,2], nums2 = [3,4]
输出：2.5
解释：合并后 [1,2,3,4]，总数4个（偶数），中间两个是 2 和 3，平均值 = 2.5。
```

---

## 最直接的思路（但太慢）

最直接的做法是：把两个数组合并成一个，排好序，然后直接取中间。

```python
merged = sorted(nums1 + nums2)
n = len(merged)
if n % 2 == 1:
    return merged[n // 2]
else:
    return (merged[n // 2 - 1] + merged[n // 2]) / 2
```

这样没问题，但时间复杂度是 O(m+n)，题目要求 O(log(m+n))。
所以需要用**二分查找**来做。

---

## 核心思路：分割线

> 这是整道题最关键的概念。理解了这个，代码就自然了。

### 第一步：理解"把数组分成左右两半"

无论数组有多长，中位数本质上就是把**合并后的数组**切一刀，让左边和右边数量相等（或左边多一个）。

```
合并后 [1, 2, 3, 4, 5]（奇数5个）

切在这里：[1, 2, 3 | 4, 5]
                   ↑
            左边3个，右边2个
            中位数 = 左边最大 = 3
```

```
合并后 [1, 2, 3, 4]（偶数4个）

切在这里：[1, 2 | 3, 4]
                ↑
          左边2个，右边2个
          中位数 = (左边最大 + 右边最小) / 2 = (2+3)/2 = 2.5
```

**规律：** 设总长 = `m+n`，左边应该有 `(m+n+1)//2` 个元素。

### 第二步：不用真正合并，在两个数组上各画一条线

现在关键来了：我们不用真的合并两个数组。

假设 nums1 和 nums2 各自画一条分割线：

```
nums1: [1, 3 | 5, 7]   ← i=2，左边取了1,3；右边有5,7
nums2: [2, 4 | 6, 8]   ← j=2，左边取了2,4；右边有6,8

合并后左边 = {1,3,2,4}，合并后右边 = {5,7,6,8}
```

如果 `i + j = (m+n+1)//2`，那么左边的元素总数就正好是中位数要求的数量。

### 第三步：什么样的分割线是"正确"的？

正确的分割线需要满足一个条件：**左边所有数 ≤ 右边所有数**。

具体来说，只需要检查两个交叉比较：

```
nums1: [... | nums1[i-1] | nums1[i] | ...]
                  ↑左边最大      ↑右边最小
nums2: [... | nums2[j-1] | nums2[j] | ...]
                  ↑左边最大      ↑右边最小

正确条件：nums1[i-1] ≤ nums2[j]
     且：nums2[j-1] ≤ nums1[i]
```

为什么只用比较这两个？因为 nums1 内部是有序的，nums2 内部也是有序的，只有这两个"跨数组"的比较才可能出问题。

---

## 图解二分过程

我们在 nums1 上做二分查找，找到让分割线"正确"的 `i`。

**例子：** `nums1 = [1, 3, 5]`，`nums2 = [2, 4, 6, 8]`

总长 = 7，左边应有 4 个元素。

---

**第1次尝试**：`i = 1`（nums1 取左边1个），则 `j = 4 - 1 = 3`（nums2 取左边3个）

```
nums1: [ 1 | 3, 5 ]
           ↑ i=1
nums2: [ 2, 4, 6 | 8 ]
                 ↑ j=3

左边：{1} + {2,4,6} = {1,2,4,6}
右边：{3,5} + {8}   = {3,5,8}

检查：nums1[i-1]=1 ≤ nums2[j]=8  ✓
     nums2[j-1]=6 ≤ nums1[i]=3  ✗  ← 6 > 3，说明 nums2 左边拿多了，i 要右移
```

所以 `i` 要增大。

---

**第2次尝试**：`i = 2`（nums1 取左边2个），则 `j = 4 - 2 = 2`

```
nums1: [ 1, 3 | 5 ]
              ↑ i=2
nums2: [ 2, 4 | 6, 8 ]
              ↑ j=2

左边：{1,3} + {2,4} = {1,2,3,4}
右边：{5} + {6,8}   = {5,6,8}

检查：nums1[i-1]=3 ≤ nums2[j]=6  ✓
     nums2[j-1]=4 ≤ nums1[i]=5  ✓  ← 找到了！
```

总长7（奇数），中位数 = 左边最大 = max(3, 4) = **4** ✓

---

## 边界情况处理

当 `i = 0` 时（nums1 左边没有元素），`nums1[i-1]` 不存在，用 `-∞` 代替。
当 `i = m` 时（nums1 全在左边），`nums1[i]` 不存在，用 `+∞` 代替。
nums2 同理。

```python
nums1_left  = nums1[i-1] if i > 0 else float('-inf')   # 左边没有就用-∞
nums1_right = nums1[i]   if i < m else float('inf')    # 右边没有就用+∞
nums2_left  = nums2[j-1] if j > 0 else float('-inf')
nums2_right = nums2[j]   if j < n else float('inf')
```

## Python 代码（带详细注释）

```python
class Solution:
    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:

        # 技巧：始终在较短的数组上二分，防止 j 越界
        if len(nums1) > len(nums2):
            nums1, nums2 = nums2, nums1

        m, n = len(nums1), len(nums2)

        # 合并后，左半边应该有多少个元素
        # 例如 m+n=7 → left_count=4；m+n=6 → left_count=3
        left_count = (m + n + 1) // 2

        # 在 [0, m] 范围内对 i 二分（i 表示 nums1 左边取几个元素）
        lo, hi = 0, m

        while lo <= hi:
            i = (lo + hi) // 2      # nums1 左边取 i 个
            j = left_count - i      # nums2 左边取 j 个

            # 获取四个关键值，越界时用无穷代替
            n1_left  = nums1[i - 1] if i > 0 else float('-inf')
            n1_right = nums1[i]     if i < m else float('inf')
            n2_left  = nums2[j - 1] if j > 0 else float('-inf')
            n2_right = nums2[j]     if j < n else float('inf')

            if n1_left <= n2_right and n2_left <= n1_right:
                # ✅ 分割线正确！
                if (m + n) % 2 == 1:
                    # 奇数：中位数 = 左边最大值
                    return float(max(n1_left, n2_left))
                else:
                    # 偶数：中位数 = (左边最大 + 右边最小) / 2
                    return (max(n1_left, n2_left) + min(n1_right, n2_right)) / 2

            elif n1_left > n2_right:
                # ❌ nums1 左边拿多了（n1_left 太大），i 要左移
                hi = i - 1
            else:
                # ❌ nums1 左边拿少了（n2_left 太大），i 要右移
                lo = i + 1

        # 正常情况不会走到这里
        raise ValueError("输入不合法")
```

## 一句话记忆法

> 在较短数组上二分找"切点" `i`，让左半边的最大值 ≤ 右半边的最小值。

## 复杂度分析

| 项目 | 值 | 原因 |
|---|---|---|
| 时间 | O(log min(m, n)) | 只在较短数组上二分 |
| 空间 | O(1) | 没有额外开辟数组 |

---

## 易错点汇总

| 错误 | 正确做法 |
|---|---|
| `i` 范围用 `[0, m-1]` | 必须是 `[0, m]`，因为"全取"或"全不取"也是合法分割 |
| 不处理边界用具体数值 | 用 `±inf` 统一处理，避免下标越界 |
| 在较长数组上二分 | 必须保证 `nums1` 是较短的，否则 `j` 可能为负 |
| 奇数时返回右边最小 | 奇数时左边多一个，中位数是**左边最大** |
| 忘记 `/ 2.0` | Python 3 中 `/` 已经是浮点除法，但要确保返回 float |
