# 二分查找总览

> [!tip] 给小白的一句话
> 二分查找的本质不是"找中间值"，而是**维护一个循环不变量**：每次都根据 mid 的信息把搜索区间缩小一半，直到区间为空或找到答案。

---

## 1. 二分查找核心：循环不变量

二分查找的所有变种都围绕一个问题：**你到底在维护什么样的区间？**

| 风格 | 区间定义 | while 条件 | left 更新 | right 更新 | 返回 |
|---|---|---|---|---|---|
| 左闭右闭 `[l, r]` | l 和 r 都在搜索范围 | `l <= r` | `l = mid + 1` | `r = mid - 1` | l 或 -1 |
| 左闭右开 `[l, r)` | l 在搜索范围，r 不在 | `l < r` | `l = mid + 1` | `r = mid` | l |

**推荐新手只用一种**：左闭右闭 `[l, r]`，因为它和数组下标 `[0, n-1]` 天然对应。

---

## 2. 二分查找四步模板

| 步骤 | 内容 |
|---|---|
| ① 定区间 | `l, r = 0, n-1`（闭区间） |
| ② 循环条件 | `while l <= r:` |
| ③ 计算 mid | `mid = l + (r - l) // 2`（防溢出） |
| ④ 缩区间 | 根据比较结果移动 l 或 r |

```python
def binary_search(nums, target):
    l, r = 0, len(nums) - 1
    while l <= r:
        mid = l + (r - l) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            l = mid + 1
        else:
            r = mid - 1
    return -1
```

## 3. 三种核心变种

### 3.1 找第一个 ≥ target 的位置（lower_bound)

```python
def lower_bound(nums, target):
    l, r = 0, len(nums) - 1
    while l <= r:
        mid = l + (r - l) // 2
        if nums[mid] >= target:
            r = mid - 1    # 收缩右边界
        else:
            l = mid + 1
    return l               # l 指向第一个 ≥ target 的位置
```

### 3.2 找最后一个 ≤ target 的位置

```python
def last_le(nums, target):
    l, r = 0, len(nums) - 1
    while l <= r:
        mid = l + (r - l) // 2
        if nums[mid] <= target:
            l = mid + 1    # 收缩左边界
        else:
            r = mid - 1
    return r               # r 指向最后一个 ≤ target 的位置
```

### 3.3 旋转数组中的二分

核心思想：**每次分两半，必有一半是有序的**。在有序的那一半中判断 target 是否在范围内。

```python
# 搜索旋转排序数组（LC33）
while l <= r:
    mid = l + (r - l) // 2
    if nums[mid] == target:
        return mid
    if nums[l] <= nums[mid]:           # 左半有序
        if nums[l] <= target < nums[mid]:
            r = mid - 1
        else:
            l = mid + 1
    else:                              # 右半有序
        if nums[mid] < target <= nums[r]:
            l = mid + 1
        else:
            r = mid - 1
```

## 4. 边界条件的底层逻辑

为什么有时返回 `l`，有时返回 `r`？

| 场景 | 返回 | 原因 |
|---|---|---|
| `lower_bound`（第一个 ≥ target） | `l` | 循环结束时 `l > r`，`l` 指向第一个 ≥ target 的位置 |
| `last ≤ target` | `r` | 循环结束时 `r < l`，`r` 指向最后一个 ≤ target 的位置 |
| `find_min_in_rotated` | `nums[l]` | 循环结束后 `l` 停在最小值处 |

**记忆技巧**：`l` 往右走，"左"小所以停在 ≥ target 的地方；`r` 往左走，"右"大所以停在 ≤ target 的地方。

## 5. 常见易错点（踩坑预警）

1. **mid 溢出**：`(l+r)//2` 在 `l+r > 2^31-1` 时会溢出。用 `l + (r - l) // 2`。
2. **死循环**：`l = mid`（而非 `mid+1`）+ `l <= r` → 在 `r = l+1` 时 `mid` 永远等于 `l`，死循环。配合 `l <= r` 时，left/right 更新必须用 `mid ± 1`。
3. **区间风格混用**：`l=0, r=n`（左闭右开）但写了 `while l <= r` → 越界。一旦选定一种区间风格，一切都要一致。
4. **旋转数组中比较对象选错**：找最小值（LC153）要跟 `nums[r]` 比较而不是 `nums[l]`。
5. **二维矩阵坐标映射**：`idx → row=idx//n, col=idx%n`，搞反就是错的。
