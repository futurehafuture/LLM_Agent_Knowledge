# 416. 分割等和子集

> 链接：https://leetcode.cn/problems/partition-equal-subset-sum/

## 题目描述

正整数数组 `nums`，判断能否将数组分成两个子集，使两个子集的元素和相等。

### 示例

```
输入：nums = [1,5,11,5]
输出：true
解释：可以分割成 [1,5,5] 和 [11]，和都是 11。
```

```
输入：nums = [1,2,3,5]
输出：false
解释：总和为 11，无法分成两个等和子集。
```

约束：`1 <= nums.length <= 200`，`1 <= nums[i] <= 100`

---

## 思路推导（DP 五步法）

### ① 前提条件

总和 `sum(nums)` 必须是偶数，否则直接返回 `False`。目标是凑出 `target = sum // 2`。

### ② 定义状态

`dp[j]` = 能否从 `nums` 中选出若干个数，使它们的和恰好为 `j`（布尔值）。

### ③ 写出转移方程

对每个数 `nums[i]`，容量为 `j` 时：

$$
dp[j] = dp[j] \;\lor\; dp[j - nums[i]]
$$

含义：不选 `nums[i]`（保持 `dp[j]`）或选 `nums[i]`（看 `dp[j-nums[i]]` 是否为 True）。

### ④ 初始化

- `dp[0] = True`（和为 0 总是可以做到——什么都不选）
- `dp[j] = False`（`j > 0`）

### ⑤ 遍历顺序

外层物品 `i` 从 0 到 `n-1`，内层容量 `j` 从 `target` 到 `nums[i]` **倒序**。

> **倒序的原因**：这是 **0/1 背包**——每个数只能选一次。倒序保证 `dp[j-nums[i]]` 是上一层（还没考虑 `nums[i]` 时）的值，不会重复选。

### ⑥ 返回值

`dp[target]`。

## Python 代码

### 写法一：0/1 背包（一维，倒序）

```python
class Solution:
    def canPartition(self, nums: list[int]) -> bool:
        total = sum(nums)
        if total % 2 != 0:
            return False
        target = total // 2

        dp = [False] * (target + 1)
        dp[0] = True

        for num in nums:
            for j in range(target, num - 1, -1):   # 倒序！
                dp[j] = dp[j] or dp[j - num]

        return dp[target]
```

### 写法二：二维 DP（直观版）

```python
class Solution:
    def canPartition(self, nums: list[int]) -> bool:
        total = sum(nums)
        if total % 2 != 0:
            return False
        target = total // 2
        n = len(nums)

        # dp[i][j] = 前 i 个数能否凑出 j
        dp = [[False] * (target + 1) for _ in range(n + 1)]
        dp[0][0] = True

        for i in range(1, n + 1):
            num = nums[i - 1]
            for j in range(target + 1):
                dp[i][j] = dp[i-1][j]  # 不选
                if j >= num:
                    dp[i][j] = dp[i][j] or dp[i-1][j - num]

        return dp[n][target]
```

### 写法三：bitset 优化

```python
class Solution:
    def canPartition(self, nums: list[int]) -> bool:
        total = sum(nums)
        if total % 2 != 0:
            return False
        target = total // 2

        # 用整数位表示可达的子集和
        bits = 1   # 第 0 位为 1，表示 sum=0 可达
        for num in nums:
            bits |= bits << num
        return (bits >> target) & 1 == 1
```

## 复杂度分析

| 写法 | 时间 | 空间 |
|---|---|---|
| 一维背包 | O(n × target) | O(target) |
| 二维 DP | O(n × target) | O(n × target) |
| bitset | O(n × target/64) | O(target/64) |

## 易错点

1. **内层必须倒序**：正序的话 `dp[j-num]` 已经被本轮更新过，意味着 `num` 被选了多次，变成了完全背包。
2. **先判 `total % 2`**：总和为奇数直接返回 False，省去后续计算。
3. **`dp[0] = True`**：和为 0（什么都不选）是基础情况。
4. **`range(target, num-1, -1)`**：注意终止条件是 `num-1` 不是 `0`，因为 `j < num` 时 `dp[j-num]` 越界。

## 知识点补充

### 0/1 背包 vs 完全背包：一个对比表

| | 0/1 背包 | 完全背包 |
|---|---|---|
| 每个物品 | 最多选 1 次 | 可选无限次 |
| 内层遍历 | **倒序** `range(W, w-1, -1)` | **正序** `range(w, W+1)` |
| 倒序原因 | 防止同一物品被重复选取 | 正序允许重复选取 |
| 代表题 | 416 分割等和子集 | [[322-零钱兑换]] |

### 一维压缩的直觉

二维 `dp[i][j]` 只依赖 `dp[i-1][...]`，所以用一维数组 + 倒序就能覆盖。倒序保证读到的还是"上一行"的值。

### bitset 技巧

用整数的二进制位表示哪些子集和是可达的。`bits << num` 相当于"所有可达值都加上 num"，`bits |= bits << num` 就是把新可达的值合并进来。非常快。
