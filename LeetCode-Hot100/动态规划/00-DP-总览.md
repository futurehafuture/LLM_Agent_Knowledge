# 动态规划总览

> [!tip] 给小白的一句话
> 动态规划本质上是**带备忘的递推**：把"原问题"拆成"子问题"，把每个子问题的答案先算出来存好，需要时直接查表，不重复计算。

---

## 1. DP 五步法（解题模板）

每道 DP 题都按这五步走，思路就清晰了：

| 步骤 | 要回答的问题 | 例（70 爬楼梯） |
|---|---|---|
| ① 定义状态 | `dp[i]` 表示什么？ | `dp[i]` = 爬到第 i 阶的方法数 |
| ② 写出转移方程 | `dp[i]` 怎么由更小的状态算来？ | `dp[i] = dp[i-1] + dp[i-2]` |
| ③ 初始化 | 边界值是多少？ | `dp[0]=1, dp[1]=1` |
| ④ 确定遍历顺序 | i 从小到大还是从大到小？正向还是反向？ | i 从 2 到 n 正向 |
| ⑤ 推导返回值 | 答案是 dp 表里的哪个值？ | `dp[n]` |

**新手最常卡的两步**：
- **状态定义**：定义错了后面全错。多想一句"我能不能用 `dp[i]` 直接算出 `dp[i+1]`"。
- **遍历顺序**：0/1 背包要倒序、完全背包要正序，搞反了答案就会偏一倍。

## 2. DP 题型分类

把 Hot 100 的 15 道 DP 题按"状态空间维度"和"问题模型"分类：

```
┌──────────────────────────────────────────────────┐
│ 一维 DP                                          │
│  ├─ 线性 DP（dp[i] 由前几个状态决定）            │
│  │   ├─ 70  爬楼梯       dp[i]=dp[i-1]+dp[i-2]  │
│  │   ├─ 198 打家劫舍     dp[i]=max(选,不选)     │
│  │   ├─ 32  最长有效括号 (复杂的线性 DP)        │
│  │   ├─ 300 LIS          dp[i]=max(dp[j])+1     │
│  │   └─ 152 乘积最大子数组 (维护双状态)         │
│  ├─ 背包 DP（容量 i + 物品/数）                  │
│  │   ├─ 322 零钱兑换     完全背包-最少数量       │
│  │   ├─ 279 完全平方数   完全背包-最少数量       │
│  │   ├─ 139 单词拆分     完全背包-能否凑出       │
│  │   └─ 416 分割等和子集 0/1 背包-能否凑出       │
│  └─ 杨辉三角形（118）：维度 1 但每行长度变化     │
│                                                  │
│ 二维 DP                                          │
│  ├─ 网格路径 DP（dp[i][j] 由相邻格子决定）       │
│  │   ├─ 62 不同路径   dp[i][j]=dp[i-1][j]+dp[i][j-1] │
│  │   └─ 64 最小路径和 dp[i][j]=grid+min(...)    │
│  ├─ 区间 DP（dp[i][j] 表示区间 [i,j] 答案）      │
│  │   └─ 5 最长回文子串 dp[i][j]=...             │
│  └─ 双串 DP（两个字符串/序列对齐）               │
│      ├─ 1143 LCS         dp[i][j]=...           │
│      └─ 72  编辑距离     dp[i][j]=...           │
└──────────────────────────────────────────────────┘
```

每一类有自己的"思维定式"，掌握模型比记题目重要。

## 3. 背包问题三件套（必看）

背包是 DP 的硬骨头，搞清三件事就过了：

### 3.1 0/1 背包：每个物品最多选一次

- **状态**：`dp[j]` = 容量 j 时的最优值
- **递推**：`dp[j] = max(dp[j], dp[j-w[i]] + v[i])`
- **遍历**：外层物品（正向），内层容量**倒序** `range(W, w[i]-1, -1)`
- **倒序的原因**：保证每个物品只被算一次（如果正序，`dp[j-w[i]]` 已经被本轮更新过，相当于物品被用了多次）

代表题：**416 分割等和子集**

### 3.2 完全背包：每个物品可以选无限次

- **状态**：`dp[j]` = 容量 j 时的最优值
- **递推**：`dp[j] = max/min(dp[j], dp[j-w[i]] + v[i])`
- **遍历**：外层物品，内层容量**正序** `range(w[i], W+1)`
- **正序的原因**：物品可以重复用，`dp[j-w[i]]` 被本轮更新过正好对应"再用一次该物品"

代表题：**322 零钱兑换、279 完全平方数、139 单词拆分**

### 3.3 排列 vs 组合：内外层顺序

| 目标 | 外层 | 内层 |
|---|---|---|
| 求**组合**数（"凑出"，顺序无关） | 物品 | 容量 |
| 求**排列**数（顺序不同算两种） | 容量 | 物品 |

> 一个 5 秒判断法：在脑子里跑一遍小例子。如果 `[1,2]` 和 `[2,1]` 算两种就是排列，算一种就是组合。

## 4. 空间优化：滚动数组

很多 DP 转移只依赖前 1-2 个状态，可以把 `O(n)` 空间压成 `O(1)`：

```python
# 朴素：dp 数组
dp = [0] * (n + 1)
dp[0], dp[1] = 1, 1
for i in range(2, n + 1):
    dp[i] = dp[i-1] + dp[i-2]
return dp[n]

# 滚动：两个变量
prev, curr = 1, 1
for _ in range(2, n + 1):
    prev, curr = curr, prev + curr
return curr
```

二维 DP（如 1143 LCS）也常能压成两行 `O(2*n)` 甚至一行 `O(n)`，但要注意**遍历方向**和**临时变量保存对角线值**。

## 5. 何时用记忆化搜索代替递推？

两种写法等价：
- **递推**（自底向上）：先算 `dp[0]`，再算 `dp[1]`...
- **记忆化搜索**（自顶向下）：从 `dp[n]` 出发递归，递归中查表/写表

用哪个？
- 状态依赖关系**清晰、规整** → 递推（更快、栈不深）
- 状态依赖关系**复杂、有大量无效状态** → 记忆化（只算用到的）
- 思路**先写出递归再加 cache** → 记忆化更直观

Python 里记忆化用 `@functools.cache`：

```python
from functools import cache

class Solution:
    def climbStairs(self, n: int) -> int:
        @cache
        def f(i):
            if i <= 1: return 1
            return f(i-1) + f(i-2)
        return f(n)
```

## 6. 常见易错点（踩坑预警）

1. **数组越界**：`dp[i-2]` 要保证 `i >= 2`，否则单独处理边界。
2. **初始化错误**：求最小值的题目 `dp` 要初始化成 `inf`，不是 0。
3. **遍历顺序搞反**：0/1 背包正序写 → 物品被用多次；完全背包倒序写 → 答案偏小。
4. **状态定义不严格**：比如 `dp[i]` 是"前 i 个的最大值"还是"以 i 结尾的最大值"，差一个字含义完全不同（300 LIS 和 53 最大子数组就是这样区分的）。
5. **结果不是 `dp[n]`**：有时答案是 `max(dp)` 或 `dp[i]+nums[i]`，要看状态定义。
6. **空间压缩后变量更新顺序**：滚动数组的 `prev, curr = curr, prev + curr` 必须用 Python 元组同时赋值，分两行写就错。

## 7. 推荐学习路径

如果你是 DP 完全新手：

```
第一周：70 爬楼梯 → 198 打家劫舍 → 118 杨辉三角 → 62 不同路径 → 64 最小路径和
        （建立"状态-转移"直觉，全是简单递推）

第二周：322 零钱兑换 → 279 完全平方数 → 139 单词拆分（吃透完全背包）
        416 分割等和子集（再吃 0/1 背包，对比正/倒序）

第三周：300 LIS → 152 乘积最大子数组（子序列/子数组类）
        1143 LCS → 72 编辑距离（双串 DP）

第四周：5 最长回文子串 → 32 最长有效括号（复杂转移）
```
