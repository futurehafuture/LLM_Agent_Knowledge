# 72. 编辑距离

> 链接：https://leetcode.cn/problems/edit-distance/

## 题目描述

两个字符串 `word1` 和 `word2`，把 `word1` 转换成 `word2` 所用的最少操作数。可用操作：插入一个字符、删除一个字符、替换一个字符。

### 示例

```
输入：word1 = "horse", word2 = "ros"
输出：3
解释：
horse → rorse (替换 h→r)
rorse → rose  (删除 r)
rose  → ros   (删除 e)
```

```
输入：word1 = "intention", word2 = "execution"
输出：5
解释：
intention → entention (替换 i→e)
entention → exention (替换 n→x)
exention → exenution (插入 u)
exenution → exection (替换 n→c)
exection → execution (插入 u)
```

约束：`0 <= word1.length, word2.length <= 500`

---

## 思路推导（DP 五步法）

### ① 定义状态

`dp[i][j]` = `word1[0..i-1]` 转换成 `word2[0..j-1]` 的最少操作数。

### ② 写出转移方程

**字符相同**（`word1[i-1] == word2[j-1]`）：

不需要操作：

```math
dp[i][j] = dp[i-1][j-1]
```

**字符不同**，三种操作取最小：

| 操作 | 含义 | 转移 |
|---|---|---|
| 删除 `word1[i-1]` | 从 `word1[0..i-2]` → `word2[0..j-1]` | `dp[i-1][j] + 1` |
| 插入 `word2[j-1]` | 从 `word1[0..i-1]` → `word2[0..j-2]` 再插一个 | `dp[i][j-1] + 1` |
| 替换 `word1[i-1]` → `word2[j-1]` | 从 `word1[0..i-2]` → `word2[0..j-2]` 再替换 | `dp[i-1][j-1] + 1` |

```math
dp[i][j] = 1 + \min(dp[i-1][j],\; dp[i][j-1],\; dp[i-1][j-1])
```

### ③ 初始化

- `dp[i][0] = i`：`word1` 的前 i 个字符全部删除才能变成空串。
- `dp[0][j] = j`：空串需要插入 j 个字符才能变成 `word2` 的前 j 个字符。

### ④ 遍历顺序

`i` 从 1 到 `m`，`j` 从 1 到 `n`，按行遍历。

### ⑤ 返回值

`dp[m][n]`。

## Python 代码

### 写法一：二维 DP

```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m, n = len(word1), len(word2)
        dp = [[0] * (n + 1) for _ in range(m + 1)]

        for i in range(m + 1):
            dp[i][0] = i
        for j in range(n + 1):
            dp[0][j] = j

        for i in range(1, m + 1):
            for j in range(1, n + 1):
                if word1[i-1] == word2[j-1]:
                    dp[i][j] = dp[i-1][j-1]
                else:
                    dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])

        return dp[m][n]
```

### 写法二：两行 DP（空间优化）

```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m, n = len(word1), len(word2)
        prev = list(range(n + 1))

        for i in range(1, m + 1):
            curr = [i] + [0] * n
            for j in range(1, n + 1):
                if word1[i-1] == word2[j-1]:
                    curr[j] = prev[j-1]
                else:
                    curr[j] = 1 + min(prev[j], curr[j-1], prev[j-1])
            prev = curr

        return prev[n]
```

### 写法三：一行 DP（极致压缩）

```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m, n = len(word1), len(word2)
        dp = list(range(n + 1))

        for i in range(1, m + 1):
            prev = dp[0]          # dp[i-1][j-1]
            dp[0] = i             # dp[i][0] = i
            for j in range(1, n + 1):
                temp = dp[j]      # 保存旧的 dp[i-1][j]
                if word1[i-1] == word2[j-1]:
                    dp[j] = prev
                else:
                    dp[j] = 1 + min(dp[j], dp[j-1], prev)
                prev = temp
        return dp[n]
```

| 写法 | 时间 | 空间 |
|---|---|---|
| 二维 DP | O(mn) | O(mn) |
| 两行 DP | O(mn) | O(n) |
| 一行 DP | O(mn) | O(n) |

## 易错点

1. **初始化 `dp[i][0]` 和 `dp[0][j]` 不能忘**：`dp[i][0] = i` 表示删 i 次，`dp[0][j] = j` 表示插 j 次。
2. **三取 min**：不匹配时有三种操作，都要考虑。
3. **匹配时不操作**：`dp[i][j] = dp[i-1][j-1]`，不是 `dp[i-1][j-1] + 0`（虽然数学上等价，但语义上"不操作"更清晰）。
4. **一行压缩时 `prev` 变量**：必须在新值覆盖 `dp[j]` 前保存旧值，用于下一轮的 `dp[i-1][j-1]`。

## 知识点补充

### 编辑距离的意义

编辑距离（Levenshtein Distance）是自然语言处理和信息检索中的经典度量，用于衡量两个字符串的相似度。变种：

| 名称 | 操作 | 公式 |
|---|---|---|
| Levenshtein | 插入、删除、替换 | 本题 |
| Damerau-Levenshtein | 插入、删除、替换、**交换** | 相邻字符交换也算一步 |

### 和 LCS 的关系

编辑距离和 LCS 有一个有趣的等式（仅限插入/删除，不含替换时）：

```math
\text{edit\_distance} = |s_1| + |s_2| - 2 \times |LCS(s_1, s_2)|
```

但本题包含替换操作，所以不直接适用。

### 双串 DP 家族对比

| 题目 | 匹配时 | 不匹配时 |
|---|---|---|
| [[1143-最长公共子序列]] | `dp[i-1][j-1] + 1` | `max(dp[i-1][j], dp[i][j-1])` |
| 72 编辑距离 | `dp[i-1][j-1]` | `1 + min(三种)` |
| LeetCode 392 判断子序列 | `j++` | `i++`（贪心） |
