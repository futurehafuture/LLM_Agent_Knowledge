# 300. 最长递增子序列

> 链接：https://leetcode.cn/problems/longest-increasing-subsequence/

## 题目描述

整数数组 `nums`，找到最长**严格递增子序列**的长度。子序列不要求连续。

### 示例

```
输入：nums = [10,9,2,5,3,7,101,18]
输出：4
解释：最长递增子序列是 [2,3,7,101] 或 [2,3,7,18]，长度为 4。
```

```
输入：nums = [0,1,0,3,2,3]
输出：4
解释：最长递增子序列是 [0,1,2,3]，长度为 4。
```

约束：`1 <= nums.length <= 2500`，`-10^4 <= nums[i] <= 10^4`

---

## 思路推导（DP 五步法）

### ① 定义状态

`dp[i]` = 以 `nums[i]` **结尾**的最长递增子序列的长度。

> 注意：是"以 i 结尾"而不是"前 i 个"。区别在于：前者的答案要取 `max(dp)`，后者的答案直接是 `dp[n-1]`。

### ② 写出转移方程

要让 `nums[i]` 接在某个 `nums[j]` 后面，需要 `nums[j] < nums[i]`：

$$
dp[i] = \max_{j < i,\; nums[j] < nums[i]} (dp[j] + 1)
$$

如果没有满足条件的 `j`，则 `dp[i] = 1`（只有自己）。

### ③ 初始化

`dp[i] = 1` for all i（每个元素自身构成长度为 1 的子序列）。

### ④ 遍历顺序

`i` 从 0 到 `n-1`，内层 `j` 从 0 到 `i-1`。

### ⑤ 返回值

`max(dp)`，不是 `dp[n-1]`！

## Python 代码

### 写法一：O(n²) DP

```python
class Solution:
    def lengthOfLIS(self, nums: list[int]) -> int:
        n = len(nums)
        dp = [1] * n
        for i in range(n):
            for j in range(i):
                if nums[j] < nums[i]:
                    dp[i] = max(dp[i], dp[j] + 1)
        return max(dp)
```

### 写法二：贪心 + 二分查找 O(n log n)

维护一个 `tails` 数组：`tails[k]` = 长度为 `k+1` 的递增子序列的最小末尾元素。

```python
import bisect

class Solution:
    def lengthOfLIS(self, nums: list[int]) -> int:
        tails = []
        for num in nums:
            pos = bisect.bisect_left(tails, num)
            if pos == len(tails):
                tails.append(num)
            else:
                tails[pos] = num
        return len(tails)
```

> `tails` 不一定是实际的 LIS，但 `len(tails)` 等于 LIS 的长度。

### 写法三：耐心排序思想（等价于写法二）

```python
class Solution:
    def lengthOfLIS(self, nums: list[int]) -> int:
        from bisect import bisect_left
        piles = []                      # 牌堆顶
        for x in nums:
            i = bisect_left(piles, x)   # 找第一个 >= x 的堆
            if i == len(piles):
                piles.append(x)         # 新堆
            else:
                piles[i] = x            # 放到该堆顶
        return len(piles)
```

| 写法 | 时间 | 空间 |
|---|---|---|
| O(n²) DP | O(n²) | O(n) |
| 贪心 + 二分 | O(n log n) | O(n) |

## 易错点

1. **返回值是 `max(dp)` 不是 `dp[n-1]`**：LIS 不一定以最后一个元素结尾。
2. **严格递增用 `<` 不是 `<=`**：`nums[j] < nums[i]`，不是 `<=`。
3. **二分用 `bisect_left`**：找"第一个 >= num 的位置"。用 `bisect_right` 会漏掉相等元素应该替换的情况。
4. **`tails` 不是答案本身**：`tails` 只能给出长度，不能还原具体子序列。

---

## 知识点补充

### 贪心 + 二分的直觉

`tails[k]` 存的是"长度为 k+1 的所有递增子序列中，末尾元素最小的那个"。末尾越小，后面越容易接上更长的序列。

每次新来一个数：
- 如果比所有 `tails` 都大 → 可以扩展更长
- 否则 → 替换掉第一个 >= 它的位置（让末尾更小）

这和扑克牌的**耐心排序**（patience sorting）完全等价。

### 如何还原具体 LIS？

需要额外维护 `prev` 指针数组。O(n²) DP 版本更方便还原路径。

### 和最长公共子序列的关系

LIS 可以转化为 LCS：将数组排序后与原数组做 LCS。但这样需要去重处理，不如直接 DP 或二分。
