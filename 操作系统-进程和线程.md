# 进程和线程

## 1. 面试官到底想考什么

“进程和线程有什么区别”表面是概念题，实际考的是你能不能把 **资源隔离**、**CPU 调度**、**地址空间**、**上下文切换**、**通信同步** 这些操作系统基础串起来。

最常见的答案是：

> 进程是资源分配的基本单位，线程是 CPU 调度的基本单位。

这句话可以作为开头，但不能只背这一句。面试官通常会继续追问：为什么进程更重、线程更轻？线程共享什么，不共享什么？进程之间怎么通信？线程之间怎么同步？多线程一定比多进程快吗？Linux 里线程到底是不是“轻量级进程”？

## 2. 零基础版解释

当你打开一个浏览器，操作系统会为它创建一个 **进程**。这个进程拥有自己的内存空间、打开的文件、网络连接等资源。它和另一个程序相互隔离。

而浏览器内部可能同时干很多事：

- 一个线程负责渲染页面
- 一个线程负责网络请求
- 一个线程负责执行 JavaScript

这些线程共享浏览器进程的资源，但能够并发执行。

因此可以简单理解为：

- 进程 = 一个公司
- 线程 = 公司里的员工

公司之间资源独立；员工共享公司资源，但各自处理不同任务。

## 3. 进程和线程的本质区别

### 3.1 资源拥有者不同

进程拥有：

- 虚拟地址空间
- 文件描述符
- Socket
- 信号处理信息
- 环境变量

线程通常不拥有这些资源，而是共享所属进程的资源。

### 3.2 调度对象不同

CPU 调度的对象是线程。

现代操作系统调度时真正切换的是线程，而不是整个进程。

因此：

- 单进程单线程 → 只能执行一个任务
- 单进程多线程 → 多任务并发
- 多进程多线程 → 更复杂并发模型

### 3.3 地址空间不同

进程之间：

- 地址空间独立
- 一个进程崩溃通常不会直接影响另一个进程

线程之间：

- 共享堆
- 共享全局变量
- 共享代码段

但每个线程有独立：

- 程序计数器（PC）
- 寄存器
- 栈（Stack）

## 4. 线程共享什么，不共享什么

### 共享

- 代码段
- 堆内存
- 全局变量
- 文件描述符
- Socket

### 不共享

- 栈
- 程序计数器
- 寄存器上下文
- 线程局部存储（TLS）

面试高频问题：

> 为什么局部变量线程安全，而全局变量不一定安全？

因为局部变量在各自线程栈中，互不影响；全局变量位于共享内存区域，多个线程同时访问时会产生竞态条件。

## 5. 为什么进程切换比线程切换重

### 进程切换

需要切换：

- 页表
- 虚拟地址空间
- TLB
- 内核上下文
- CPU 寄存器

代价较大。

### 线程切换

通常只需要切换：

- 栈
- PC
- 寄存器

不需要切换整个地址空间。

因此线程切换成本通常低于进程切换。

## 6. 通信方式

### 进程通信（IPC）

常见方式：

- Pipe（管道）
- FIFO
- Message Queue（消息队列）
- Shared Memory（共享内存）
- Socket
- Signal

面试最爱问：

> 哪种 IPC 最快？

答案通常是共享内存，因为减少了数据拷贝。

### 线程通信

线程天然共享内存。

主要问题不是通信，而是同步。

常见同步机制：

- Mutex
- RWLock
- Semaphore
- Condition Variable
- Atomic

## 7. Linux 里的线程是什么

很多教材把线程和进程分得很开。

但 Linux 内核里：

> 线程本质上也是进程。

Linux 使用 task_struct 描述执行实体。

通过 clone() 系统调用决定共享哪些资源：

- 共享地址空间 → 线程
- 不共享地址空间 → 进程

因此常说：

> Linux 线程 = 轻量级进程（LWP）

## 8. 面试标准答案（1分钟版）

进程是资源分配的基本单位，线程是 CPU 调度的基本单位。

进程拥有独立的地址空间和系统资源，进程之间相互隔离；线程运行在进程内部，共享进程的代码段、堆、全局变量和文件描述符等资源，但拥有独立的栈、程序计数器和寄存器上下文。

由于线程共享地址空间，因此创建和切换成本通常低于进程；而进程隔离性更好，稳定性更高。

进程之间需要通过 IPC 机制通信，线程之间可以直接访问共享内存，但需要使用锁、信号量等同步机制解决并发安全问题。

在 Linux 中，线程本质上是通过 clone() 创建的轻量级进程。
