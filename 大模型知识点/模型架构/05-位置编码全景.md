# 位置编码全景：从 Sinusoidal 到 RoPE / ALiBi / YaRN

> 核心问题：Transformer 为什么需要位置编码？位置编码有哪几大类？各自的优缺点？面试问"位置编码有哪些"该怎么答？

---

## 0. 面试怎么答（30 秒电梯版）

> "Transformer 的 self-attention 是 **置换不变**的（顺序不影响输出），所以必须显式注入位置信息。
>
> 位置编码大致分两条主线和五代演进：
>
> **绝对位置编码（Absolute）**：把位置编成向量加到 token embedding 上
> - **Sinusoidal**（Transformer 原版）：$`\sin/\cos`$ 不同频率组合，免训练、可外推但弱
> - **Learned**（BERT/GPT-2）：直接学一张 $`[max\_len, d]`$ 的表，效果好但不能超过训练长度
>
> **相对位置编码（Relative）**：把位置差 $`i - j`$ 编进 attention 分数
> - **Transformer-XL / T5 Relative Bias**：手动在 score 上加偏置
> - **ALiBi**：纯线性偏置 $`-m \cdot (i-j)`$，硬件友好，外推优秀，被 BLOOM 用
>
> **RoPE 旋转位置编码（现代主流）**：通过旋转矩阵把绝对位置编进 Q/K，注意力分数天然包含相对位置 → **既是绝对编码、效果像相对编码**。LLaMA、Qwen、GLM 都用。
>
> **RoPE 长文本扩展**：Position Interpolation / NTK-aware / YaRN / Dynamic NTK，解决"训练 8k 用到 128k"的外推问题。"

记忆树：

```
位置编码
├── 绝对位置编码
│   ├── Sinusoidal (Attention is All You Need)
│   └── Learned (BERT, GPT)
├── 相对位置编码
│   ├── Transformer-XL
│   ├── T5 Relative Bias
│   └── ALiBi (BLOOM, MPT)
└── 旋转位置编码（折中：绝对形式，相对效果）
    ├── RoPE (LLaMA, Qwen, GLM, DeepSeek)
    └── RoPE 外推方案
        ├── Position Interpolation (PI)
        ├── NTK-aware
        ├── Dynamic NTK
        └── YaRN
```

## 1. 为什么需要位置编码

Self-attention 公式：

```math
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V
```

如果把输入 token 顺序打乱，$`Q, K, V`$ 三个矩阵的行也跟着打乱，但**输出对应行也跟着打乱**——结果不变只是位置变了。

**也就是说，Transformer 不知道 token 之间的顺序。**

而对自然语言，"我打你"和"你打我"完全不同。所以必须**显式注入位置信号**。

> 对比 RNN（自带顺序）、CNN（有局部位置归纳偏置），Transformer 是位置无关的。

## 2. 绝对位置编码（APE: Absolute Position Encoding）

### 2.1 Sinusoidal 正弦位置编码（Transformer 原版）

#### 公式

```math
PE(pos, 2i) = \sin\left(\frac{pos}{10000^{2i/d}}\right),\quad PE(pos, 2i+1) = \cos\left(\frac{pos}{10000^{2i/d}}\right)
```

- $`pos`$：token 在序列中的位置（0, 1, 2, ...）
- $`i`$：embedding 维度索引
- $`d`$：embedding 总维度
- 不同维度对应不同频率（$`10000^{2i/d}`$ 从快到慢）

#### 用法

直接加到 token embedding 上：

```math
x_t = \text{Embed}(token_t) + PE(t)
```

#### 直觉

低维分量频率高（变化快）→ 区分相近位置；高维分量频率低（变化慢）→ 区分远距离位置。

#### 优点
- **不需要学习**：免参数
- **可外推**（理论上）：公式对任意 $`pos`$ 都有定义
- 不同位置的编码可以通过**线性组合**得到其他位置（隐含相对位置信息）

#### 缺点
- 直接加在 embedding 上，**位置和语义混在一起**，干扰语义表达
- 实际外推效果不好（长度泛化弱）

### 2.2 Learned Position Embedding（BERT / GPT-2 / GPT-3）

#### 做法

学一张 $`[max\_len, d]`$ 的查找表，每个位置对应一个**可学习向量**：

```python
self.pos_embed = nn.Embedding(max_len, d_model)
x = token_embed + self.pos_embed(position_ids)
```

#### 优点
- 实现极简
- 训练充分后效果好

#### 缺点
- **完全不能外推**：训练 $`max\_len = 512`$，推理时给 1000 位置直接报错或乱码
- 占参数（虽然只有 $`max\_len \cdot d`$）

## 3. 相对位置编码（RPE: Relative Position Encoding）

### 3.1 动机

人类语言对**绝对位置**不敏感（"第 5 个词"没意义），对**相对位置**敏感（"上一个词"、"5 个词之前"有意义）。

相对位置编码直接把"位置差 $`i - j`$"编进注意力。

### 3.2 Transformer-XL Style

把 $`QK^T`$ 展开：

```math
A_{ij} = (E_i + U_i)^T W_q^T W_k (E_j + U_j)
```

其中 $`E`$ 是 token embedding，$`U`$ 是位置编码。Transformer-XL 把展开后的项**重新参数化**，引入相对位置 $`R_{i-j}`$ 替代绝对位置 $`U_j`$。

### 3.3 T5 Relative Position Bias

T5 的做法极简：直接在 attention score 上加一个**位置差偏置**：

```math
\text{score}_{ij} = \frac{Q_i K_j^T}{\sqrt{d}} + b(i - j)
```

$`b(i-j)`$ 是一个**可学习的标量表**，按位置差取值（远距离会被 bucket 化）。

### 3.4 ALiBi（Attention with Linear Biases）

#### 公式

```math
\text{score}_{ij} = \frac{Q_i K_j^T}{\sqrt{d}} - m_h \cdot |i - j|
```

- $`m_h`$ = 每个 attention head 有一个固定斜率（不可学习），按几何级数分布
- 距离越远，被减得越多 → 自然衰减远距离 attention

#### 优点
- **完全不需要位置 embedding**
- **极强的外推能力**：训练 2k 用到 16k 几乎不掉点
- 实现简单（attention 加个常数）
- 硬件友好

#### 缺点
- "线性衰减"假设可能过于简单
- 不是当前 SOTA 模型主流（被 RoPE 取代）

#### 用户
- BLOOM
- MPT
- Falcon

## 4. RoPE：旋转位置编码（现代主流）

### 4.1 核心思想

通过**旋转矩阵**对 Q、K 进行位置相关的旋转。详见 [[04-RoPE-旋转位置编码]]。

二维情形下，位置 $`t`$ 处的 Q 被旋转角度 $`t\theta`$：

```math
R(t\theta) = \begin{pmatrix} \cos(t\theta) & -\sin(t\theta) \\ \sin(t\theta) & \cos(t\theta) \end{pmatrix}
```

应用到 Q 和 K 上：

```math
Q'_m = R(m\theta) Q_m,\quad K'_n = R(n\theta) K_n
```

内积：

```math
Q'_m \cdot K'_n = Q_m^T R(m\theta)^T R(n\theta) K_n = Q_m^T R((n-m)\theta) K_n
```

—— **注意力分数只依赖相对位置 $`n - m`$**。

### 4.2 高维实现

把 $`d`$ 维的 Q/K 切成 $`d/2`$ 个二维子空间，每对维度用不同的旋转频率：

```math
\theta_i = \text{base}^{-2i/d},\quad i = 0, 1, \dots, d/2 - 1
```

base 通常是 10000（原版）或 1000000（Qwen3、长文本模型）。

### 4.3 为什么 RoPE 取代了其他

| 维度 | RoPE 优势 |
|---|---|
| 位置注入方式 | 直接乘到 Q/K 上，不破坏语义 |
| 相对位置 | 天然蕴含（数学结构决定） |
| 外推 | 经过 PI/NTK/YaRN 等扩展极强 |
| 兼容性 | 与 Flash Attention 等加速兼容 |
| 工程 | 实现简单 |

## 5. RoPE 长文本扩展方案

训练时长度 $`L_\text{train}`$，推理想用 $`L > L_\text{train}`$，直接外推 RoPE 会**效果断崖**（高频维度看到没训练过的旋转角度）。

### 5.1 Position Interpolation（PI, Meta 2023）

把推理位置**等比例缩到训练范围**：

```math
\theta_i' = \theta_i \cdot \frac{L_\text{train}}{L}
```

```python
# 原版
freqs = 1.0 / (base ** (torch.arange(0, dim, 2) / dim))
# PI 版
freqs = 1.0 / (base ** (torch.arange(0, dim, 2) / dim)) * (L_train / L_target)
```

效果：**通过少量 fine-tuning** 可以把 LLaMA-2 7B 从 4k 扩到 32k。

### 5.2 NTK-aware（Reddit 网友 bloc97 提出）

**观察**：直接 PI 会让高频部分被压缩太多，损失精度。

**做法**：不缩放频率，而是**改 base**：

```math
\text{base}' = \text{base} \cdot \left(\frac{L}{L_\text{train}}\right)^{d/(d-2)}
```

高频维度受影响小，低频维度被有效"拉远"。

**效果**：**不需要 fine-tuning** 就能扩展（zero-shot 外推）。

### 5.3 Dynamic NTK

NTK-aware 用固定的 base，但推理时长度可变。Dynamic NTK 让 base **随实际推理长度动态调整**：

```math
\text{base}'(L_\text{cur}) = \text{base} \cdot \max\left(1, \frac{L_\text{cur}}{L_\text{train}}\right)^{d/(d-2)}
```

短上下文用原 base，长上下文自动调整。

### 5.4 YaRN（Yet another RoPE extensioN）

YaRN 综合了 PI 和 NTK，对**不同频率维度差异化处理**：

- **高频维度**（变化快，看局部细节）：不动 → 保留精度
- **低频维度**（变化慢，看全局）：插值 → 拉长
- **中频维度**：渐变过渡

YaRN 还引入 **attention 温度调整**：

```math
\text{score}_{ij} = \frac{Q_i K_j^T}{t \cdot \sqrt{d}}
```

其中 $`t = 0.1 \ln(L/L_\text{train}) + 1`$，补偿插值后 attention 分布变平的问题。

**效果**：Qwen2/Qwen3、Mistral 长上下文版本都用 YaRN。

### 5.5 DCA（Dual Chunk Attention）

不是改 RoPE 本身，而是**分块处理**长上下文：
- 块内：正常 RoPE
- 块间：用相对块编号代替绝对位置

Qwen3 长文档场景结合 YaRN + DCA。

## 6. 各类位置编码对比表

| 方法 | 类型 | 注入方式 | 外推 | 是否学习 | 代表模型 |
|---|---|---|---|---|---|
| **Sinusoidal** | 绝对 | + token emb | 弱 | ❌ | Transformer 原版 |
| **Learned** | 绝对 | + token emb | ❌ | ✅ | BERT, GPT-2, GPT-3 |
| **Transformer-XL** | 相对 | 修改 attention | 中 | 部分 | Transformer-XL |
| **T5 RB** | 相对 | + attention 偏置 | 中 | ✅（标量表） | T5 |
| **ALiBi** | 相对 | - 距离偏置 | 强 | ❌ | BLOOM, MPT |
| **RoPE** | 绝对/相对（混合） | 旋转 Q/K | 中（需扩展） | ❌ | LLaMA, Qwen, DeepSeek |
| **RoPE+PI** | 同 RoPE | 缩频率 | 强（需微调） | ❌ | LLaMA-2 long |
| **RoPE+NTK** | 同 RoPE | 改 base | 强（zero-shot） | ❌ | CodeLLaMA |
| **RoPE+YaRN** | 同 RoPE | 频率分段处理 | 极强 | ❌ | Qwen2/3, Mistral |

## 8. 面试高频追问

### Q1：位置编码加在 token embedding 上和加在 attention score 上有什么区别？
- 加在 embedding 上（Sinusoidal/Learned）：每层 attention 都受位置影响，但**位置和语义信息纠缠**
- 加在 attention score 上（T5/ALiBi）：位置和语义解耦，但每个 head/layer 都需要重新引入
- 乘到 Q/K 上（RoPE）：**位置以乘法形式参与**，attention 分数自然带相对位置

### Q2：为什么 RoPE 既是绝对位置编码又是相对位置编码？
**形式上是绝对的**：每个位置 $`t`$ 对应一个旋转 $`R(t\theta)`$。
**效果上是相对的**：内积 $`Q'_m \cdot K'_n`$ 只依赖 $`n - m`$。
这是 RoPE 的精髓——**用绝对的实现方式，达到相对的效果**。

### Q3：ALiBi 为什么外推强而 RoPE 直接外推弱？
- ALiBi：偏置是 $`-m \cdot |i - j|`$，距离越远衰减越多，**没有训练过的距离也能合理处理**
- RoPE：高频维度旋转角度超过训练范围时，对应的 $`\cos/\sin`$ 值未见过，attention 分布偏离

### Q4：base 改大有什么效果？
增大 base = 让低频维度更慢变化 = 能编码更长的有效距离。Qwen3 用 base = 1M 就是为长上下文。

### Q5：YaRN 比 NTK-aware 强在哪？
NTK-aware 一刀切改 base，对所有维度无差别。YaRN **按频率分组差异化处理**：
- 高频（细节）：不动
- 低频（全局）：插值
- 中频：渐变
同时调 attention 温度补偿分布变平。

### Q6：能不能同时用 Sinusoidal + RoPE？
理论可以，实践没有意义。RoPE 已经是更好的设计。

### Q7：为什么 BERT 不用 RoPE？
BERT (2018) 早于 RoPE (2021)。如果现在重做 BERT 必然用 RoPE。

### Q8：相对位置编码的"绝对位置无关"是优点还是缺点？
对一般文本是优点（句首句尾不应特别）。但对某些任务（如代码缩进、表格）绝对位置有意义，所以现代模型偏向 **绝对形式 + 相对效果** 的 RoPE。

见 [[06-ViT与多模态位置编码]]。
