# LeetCode 33. 搜索旋转排序数组

> 整数数组 `nums` 按升序排列，数组中的值**互不相同**。在传递给函数之前，`nums` 在某个下标处发生了**旋转**（如 `[0,1,2,4,5,6,7]` 可能变为 `[4,5,6,7,0,1,2]`）。给你 `nums` 和整数 `target`，如果在数组中存在 `target`，返回它的下标，否则返回 `-1`。
>
> 你必须设计 **O(log n)** 的解法。
>
> - 输入：`nums = [4,5,6,7,0,1,2], target = 0` → 输出：`4`
> - 输入：`nums = [4,5,6,7,0,1,2], target = 3` → 输出：`-1`
> - 输入：`nums = [1], target = 0` → 输出：`-1`

---

## 0. 面试怎么答（30 秒电梯版）

> "数组旋转后整体不再有序，但**关键性质**是：从中点切开，**左右两段必有一段是完全有序的**。
>
> 算法：标准二分。每次算 `mid`，判断 `nums[left..mid]` 是否有序（看 `nums[left] <= nums[mid]`）：
> - 如果**左半有序**：判断 target 是否在 `[nums[left], nums[mid])` 内，是就去左半，否则去右半
> - 如果**右半有序**：判断 target 是否在 `(nums[mid], nums[right]]` 内，是就去右半，否则去左半
>
> 时间 O(log n)、空间 O(1)。
>
> **边界要小心**：`<=` vs `<`、是否包含端点、数组只有一个元素时。"

---

## 1. 核心观察：旋转后的有序性

旋转数组示例：`[4, 5, 6, 7, 0, 1, 2]`

```
索引：  0  1  2  3  4  5  6
值：    4  5  6  7  0  1  2
        └─有序段─┘  └有序段┘
                  ↑
              旋转点
```

任取一个中点 `mid`：
- 若 `mid = 3`（值 7）：左半 `[4,5,6,7]` 有序，右半 `[0,1,2]` 也有序，**两段都有序**
- 若 `mid = 1`（值 5）：左半 `[4, 5]` 有序，右半 `[6, 7, 0, 1, 2]` **包含旋转点不有序**
- 若 `mid = 5`（值 1）：左半 `[4, 5, 6, 7, 0, 1]` **包含旋转点不有序**，右半 `[1, 2]` 有序

**关键引理**：在旋转数组中切一刀，**至少有一段是完全有序的**。

判断左半 `nums[left..mid]` 是否有序的依据：

$$
\text{nums[left]} \le \text{nums[mid]} \Rightarrow \text{左半有序}
$$

否则右半 `nums[mid..right]` 必有序。

---

## 2. 算法步骤

```
1. left = 0, right = n - 1
2. while left <= right:
     mid = (left + right) // 2
     if nums[mid] == target: return mid

     if nums[left] <= nums[mid]:      ← 左半有序
         if nums[left] <= target < nums[mid]:    ← target 在左半
             right = mid - 1
         else:
             left = mid + 1
     else:                              ← 右半有序
         if nums[mid] < target <= nums[right]:    ← target 在右半
             left = mid + 1
         else:
             right = mid - 1
3. return -1
```

---

## 3. 代码

```python
class Solution:
    def search(self, nums: list[int], target: int) -> int:
        left, right = 0, len(nums) - 1

        while left <= right:
            mid = (left + right) // 2

            if nums[mid] == target:
                return mid

            # 判断左半 [left..mid] 是否有序
            if nums[left] <= nums[mid]:
                # 左半有序
                if nums[left] <= target < nums[mid]:
                    right = mid - 1
                else:
                    left = mid + 1
            else:
                # 右半 [mid..right] 有序
                if nums[mid] < target <= nums[right]:
                    left = mid + 1
                else:
                    right = mid - 1

        return -1
```

复杂度：
- 时间 $O(\log n)$（每次砍半）
- 空间 $O(1)$

## 4. Walk-through 示例

`nums = [4, 5, 6, 7, 0, 1, 2]`, `target = 0`

| 步骤 | left | right | mid | nums[mid] | 判断 | 操作 |
|---|---|---|---|---|---|---|
| 1 | 0 | 6 | 3 | 7 | 左半 `[4,5,6,7]` 有序 (4 ≤ 7)，target 0 不在 [4, 7) | left = 4 |
| 2 | 4 | 6 | 5 | 1 | 左半 `[0,1]` 有序 (0 ≤ 1)，target 0 在 [0, 1) | right = 4 |
| 3 | 4 | 4 | 4 | 0 | 命中！返回 4 ✅ |

## 5. 边界细节

### 5.1 `nums[left] <= nums[mid]` 用 `<=` 不是 `<`

考虑 `nums = [3, 1], left = 0, mid = 0`：
- `nums[left] = nums[mid] = 3`
- 用 `<=`：判定为"左半有序"（确实，单元素必有序），target 不在 `[3, 3)` → 去右半 ✅
- 用 `<`：判定为"右半有序"，按右半逻辑也能对，但**左半永远不会判有序**，少数情况可能漏判

**Pythonic 经验**：`<=` 是更稳的写法，覆盖 `left == mid` 的退化情况。

### 5.2 target 边界用 `<` 还是 `<=`

```python
if nums[left] <= target < nums[mid]:
    right = mid - 1
```

- **下界 `<=` nums[left]**：因为 target 可能正好等于 left
- **上界 `< nums[mid]`**：因为 mid 已经在前面判过，target ≠ nums[mid]

### 5.3 `while left <= right` 还是 `left < right`

用 `<=` 配合 `right = mid - 1` / `left = mid + 1` 是标准的"闭区间"二分。

闭区间 `[left, right]` 表示"还要找的范围"，当 left > right 时区间为空，无解。

### 5.4 整数溢出

`mid = (left + right) // 2` 在 Python 不会溢出。在 C++/Java 中要写：

```cpp
int mid = left + (right - left) / 2;
```

## 6. 变体：包含重复元素（LC 81）

如果数组**允许重复**（如 `[1, 0, 1, 1, 1]`），上面算法挂掉，因为：

- `nums = [1, 0, 1, 1, 1], left = 0, mid = 2, right = 4`
- `nums[left] = nums[mid] = nums[right] = 1`
- **没法判断左右哪边有序**

**解决**：遇到 `nums[left] == nums[mid] == nums[right]` 时，**两端各前进一步**：

```python
class Solution:
    def search(self, nums: list[int], target: int) -> bool:
        left, right = 0, len(nums) - 1
        while left <= right:
            mid = (left + right) // 2
            if nums[mid] == target:
                return True

            # 关键：处理重复
            if nums[left] == nums[mid] == nums[right]:
                left += 1
                right -= 1
                continue

            if nums[left] <= nums[mid]:
                if nums[left] <= target < nums[mid]:
                    right = mid - 1
                else:
                    left = mid + 1
            else:
                if nums[mid] < target <= nums[right]:
                    left = mid + 1
                else:
                    right = mid - 1
        return False
```

**最坏复杂度退化为 O(n)**（如 `[1, 1, 1, 1, ..., 1]`），但**平均仍是 O(log n)**。

## 7. 变体：找旋转点（LC 153 / 154）

LC 153（无重复）：找最小值的下标。

```python
def findMin(nums):
    left, right = 0, len(nums) - 1
    while left < right:
        mid = (left + right) // 2
        if nums[mid] > nums[right]:
            # 最小值在右半（开区间）
            left = mid + 1
        else:
            # 最小值在左半（包含 mid）
            right = mid
    return nums[left]
```

LC 154（有重复）：和 LC 81 类似，遇到 `nums[mid] == nums[right]` 时 `right -= 1`。

## 8. 其他思路（不推荐但要知道）

### 8.1 先二分找旋转点再二分找 target

1. 用 LC 153 的方法找到旋转点 `pivot`
2. 判断 target 在哪一半
3. 在对应半段做标准二分

**两次二分，常数大，代码长**，不如一次二分。

### 8.2 直接 O(n) 遍历

```python
for i, v in enumerate(nums):
    if v == target: return i
return -1
```

不满足 O(log n) 要求，**面试拿不到分**。

## 9. 面试高频追问

### Q1：为什么旋转数组也能二分？
关键性质：**从任意中点切开，至少一边是完全有序的**。二分通过判断"哪边有序 + target 是否在有序段内"来决定缩到哪一半。

### Q2：为什么用 `nums[left] <= nums[mid]` 而不是 `nums[left] < nums[mid]`？
处理 `left == mid` 的退化情形（数组只剩 1-2 个元素）。`<=` 时把单元素也判为"有序"，逻辑统一。

### Q3：如果数组没旋转怎么办？
没旋转 = 旋转 0 位 = `nums[left] <= nums[mid]` 一直成立 → 算法退化为标准二分，仍然正确。

### Q4：复杂度怎么证明？
每次循环中 `right - left` 至少减半（因为去掉了 mid 那侧），所以 $O(\log n)$ 次循环。

### Q5：包含重复元素时为什么会退化？
当 `nums[left] == nums[mid] == nums[right]` 时无法判断有序段，被迫两端各走一步，最坏情况（全相同元素）变成 O(n)。

### Q6：能不能不用二分？
能（线性扫描），但**不满足 O(log n) 要求**。面试这道题就是考二分思维。

### Q7：你是怎么记住四种情况的？
两层判断：
1. **哪边有序**（用 `nums[left] vs nums[mid]`）
2. **target 在不在有序段内**（用 target 和段端点比较）

然后**去 target 所在的那一半**：
- 有序段内 → 缩进有序段
- 不在有序段 → 缩到另一半（即使另一半不有序，下一轮继续切）

### Q8：如果让你写一个通用框架，能不能套用到 LC 81/153/154？
可以。核心还是"切一半，找有序段，看 target/最小值在哪边"。LC 81 加重复处理，LC 153/154 求位置不是 target。

## 10. 完整可运行测试

```python
def test():
    sol = Solution()
    cases = [
        ([4,5,6,7,0,1,2], 0, 4),
        ([4,5,6,7,0,1,2], 3, -1),
        ([1], 0, -1),
        ([1], 1, 0),
        ([1,3], 3, 1),
        ([3,1], 1, 1),
        ([5,1,3], 3, 2),
        ([1,2,3,4,5], 3, 2),     # 未旋转
        ([5,4,3,2,1], 3, -1),    # 注意原题保证升序+旋转，这个不合法但测健壮性
    ]
    for nums, target, expected in cases:
        got = sol.search(nums, target)
        status = "✅" if got == expected else "❌"
        print(f"{status} {nums} target={target} → {got} (expected {expected})")

test()
```
