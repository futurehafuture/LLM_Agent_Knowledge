# 多模态模型推理

> 多模态服务不仅多了一种输入格式，还增加了媒体下载/解码、视觉或音频 encoder、可变数量的模态 token 和新的缓存、调度问题。

## 1. 典型流水线

```text
图片/视频/音频
  → 下载与安全校验
  → resize/sample/feature extraction
  → Vision/Audio Encoder
  → Projector/Resampler
  → modality embeddings/tokens
  → LLM Prefill
  → LLM Decode
```

CPU 预处理、encoder 和 LLM 可能是三个独立瓶颈。

## 2. 图像 Token 数量

固定分辨率 ViT 将图像切成 patch。若图像高宽为 $`H,W`$，patch size 为 $`P`$：

```math
N_{patch}=\frac HP\cdot\frac WP.
```

动态分辨率、AnyRes 或视频会让 token 数大幅变化。多模态“请求数相同”不代表工作量相同，调度应按像素、帧数、encoder token 和 LLM token 预算。

## 3. Encoder 与 LLM 的性能差异

- Encoder 常一次处理整张图/多帧，偏 compute-bound；
- LLM Prefill 处理视觉 token + 文本 token；
- Decode 与纯文本模型类似，但 KV 前缀更长。

因此可能需要独立 encoder batch、encoder cache 或 encode/decode 分离资源池。

## 4. Multimodal Cache

同一图片被多次提问时，可缓存：

- 下载后的 bytes；
- 预处理 tensor；
- encoder embeddings；
- projector 输出；
- 完整 LLM prefix KV。

越靠后缓存，跳过的计算越多，但与模型版本、processor 配置和 prompt template 耦合越强。

## 5. Cache Key

至少包含：

```math
key=H(media\ bytes,\ processor,\ encoder,\ projector,\ model\ version,\ config).
```

不能只按 URL 缓存，因为 URL 内容可能变化；也不能忽略 resize、crop、frame sampling 等参数。

## 6. Dynamic Batching

不同图片尺寸与视频帧数造成 ragged input。常见策略：

- 按分辨率/帧数 bucket；
- 限制每 batch 总像素或总视觉 token；
- padding 到 bucket shape；
- encoder 与 LLM 分别 batching；
- 对超大媒体做准入或降采样。

## 7. 视频推理

视频增加：

- 解码与帧采样成本；
- 时间维位置编码；
- 大量视觉 token；
- 长上下文 KV；
- 上传与网络延迟。

应明确 FPS、最大帧数、分辨率和采样算法，否则 benchmark 不可比较。

## 8. Encoder-Decoder 分离

可把视觉 encoder 放在独立 GPU 池，embedding 传给 LLM 池。相比 PD 分离，传输的是 encoder embedding 而不是所有层 KV，大小与模态 token 数和 hidden size 有关。

分离适合 encoder 与 LLM 负载比例变化大、可独立 batching 或使用不同硬件的场景。

## 9. 安全与资源控制

- 限制图片像素、视频时长、压缩炸弹；
- 下载域名、大小与超时；
- 媒体解码 sandbox；
- 对隐藏文本、提示注入做上层防护；
- 失败时释放 encoder/LLM 预留资源。

## 10. 评测

分开记录：

- media download/decode；
- preprocessing；
- encoder latency；
- LLM TTFT/TPOT；
- 端到端延迟；
- 不同分辨率、图片数、帧数；
- multimodal cache hit rate。

## 11. 面试回答

> 多模态请求在 LLM 前增加媒体处理、encoder 和 projector，并产生可变数量的视觉/音频 token。调度不能只按文本长度，应按像素、帧数、encoder token 和 LLM token 预算。可缓存 encoder embedding 或完整 prefix KV，也可把 encoder 与 LLM 分离部署；cache key 必须包含媒体内容、processor 和模型版本。
