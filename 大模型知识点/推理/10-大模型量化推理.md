# 大模型量化推理

> 量化把高精度数映射到更少 bit。它可能降低容量和带宽，但只有硬件与 kernel 真正支持该格式时，才会转化为速度提升。

## 1. 基本量化公式

对称均匀量化：

```math
q=\mathrm{clip}\left(\mathrm{round}\left(\frac{x}{s}\right),q_{min},q_{max}\right),\qquad \hat x=sq.
```

非对称量化加入 zero point $`z`$：

```math
q=\mathrm{round}(x/s)+z,\qquad \hat x=s(q-z).
```

scale 可以按 tensor、channel 或 group 计算。粒度越细，精度通常越好，但元数据和 kernel 复杂度更高。

## 2. 先看量化对象

| 记法 | 权重 | 激活 | 典型目的 |
|---|---|---|---|
| W4A16 | INT4 | FP16/BF16 | 降低权重容量和 decode 带宽 |
| W8A8 | INT8 | INT8 | 使用整数 Tensor Core 加速 GEMM |
| FP8 | FP8 | FP8/高精度累加 | 数据中心低精度计算 |
| KV INT8/FP8/INT4 | 不变 | KV 单独量化 | 长上下文和大 batch |

权重、激活和 KV 的分布不同，不能用一种算法统一处理。

## 3. PTQ 与 QAT

- **PTQ**：训练后用校准数据量化，成本低；GPTQ、AWQ、SmoothQuant 属于这一大类。
- **QAT**：训练时模拟量化误差，让模型适应低精度，成本高但极低 bit 下通常更稳。

## 4. GPTQ

GPTQ 逐列量化权重，并用近似 Hessian 信息补偿误差。核心目标是让层输出误差而非单个权重误差尽量小：

```math
\min_{\hat W}\|WX-\hat WX\|_2^2.
```

优点是低 bit 精度较好；缺点是量化过程和格式可能与具体推理 kernel 强绑定。

## 5. AWQ

AWQ 观察到少量与激活相关的显著权重对输出很重要，通过 per-channel scaling 保护这些权重，再进行 weight-only 量化。它不需要重训练，适合 W4A16 部署。

## 6. SmoothQuant

激活中的 outlier 使 W8A8 困难。SmoothQuant 利用线性层等价变换，把一部分激活难度迁移到权重：

```math
XW=(XS^{-1})(SW).
```

选择对角缩放矩阵 $`S`$ 后，激活更“平滑”，更容易 INT8 量化，而权重通常更能承受扩大后的动态范围。

## 7. FP8、MXFP 与 NVFP4

浮点格式保留指数，动态范围比定点更灵活：

- FP8 常见 E4M3、E5M2；
- MX 格式为小块共享 scale；
- NVFP4 等格式依赖新硬件的块缩放和低精度 Tensor Core。

格式名称相同不代表累加精度、scale 粒度和硬件路径相同，比较时必须说明完整 recipe。

## 8. KV Cache 量化

KV 每个 token 都会长期保存，量化收益随 batch 和上下文增长。难点包括：

- K 的通道 outlier；
- 每 token/per-channel scale 的元数据；
- 写 cache 时量化、读 cache 时反量化；
- decode attention kernel 是否原生支持低精度 KV。

若反量化开销超过节省的带宽，容量下降也未必带来低延迟。

## 9. 精度评测

不能只测 perplexity。还应覆盖：

- 数学、代码、长上下文检索；
- 多语言与低频 token；
- instruction following；
- 生成长度与重复率；
- 不同校准集的泛化。

## 10. 为什么模型变小却不一定更快

1. GPU 没有对应低精度指令；
2. kernel 先反量化再用 FP16 GEMM；
3. batch 太大后系统转为 compute-bound；
4. 量化格式造成不规则内存布局；
5. 其他算子和通信成为瓶颈。

## 11. 选型思路

| 目标 | 常见起点 |
|---|---|
| 单卡放下更大模型 | W4A16 AWQ/GPTQ |
| 数据中心高吞吐 | FP8 或成熟 W8A8 |
| 长上下文大并发 | KV FP8/INT8，再评估 INT4 |
| CPU/端侧 | GGUF、INT4/INT8 与硬件专用 kernel |

## 12. 面试回答

> GPTQ 用二阶信息逐步补偿权重量化误差；AWQ 用激活识别并保护显著权重；SmoothQuant 通过等价缩放把激活 outlier 的难度迁移到权重，支持 W8A8。量化能否加速取决于瓶颈和 kernel，不是文件大小降为四分之一就必然快四倍。
