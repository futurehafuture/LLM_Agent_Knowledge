# 长上下文推理优化

> 长上下文同时放大 Prefill 计算、KV Cache 容量、Decode KV 带宽和调度阻塞，不是只把 `max_model_len` 调大。

## 1. 四个线性或平方增长项

对于长度 $`T`$：

- Prefill attention FLOPs：标准 dense attention 近似 $`O(T^2d)`$；
- KV Cache：$`O(TLd_{kv})`$；
- 每步 decode attention：$`O(Td_{kv})`$；
- 首 token 延迟与长 prefill 调度时间显著增加。

FlashAttention 降低 IO 和中间内存，但不消除 dense attention 的 $`T^2`$ 运算。

## 2. Sliding Window Attention

只保留最近 $`W`$ 个 token：

```math
\operatorname{Attn}(q_t,K_{\max(1,t-W+1):t},V_{\max(1,t-W+1):t}).
```

KV 和 decode attention 成本封顶在窗口大小，但模型无法直接访问窗口外信息。它通常需要模型训练时就支持。

## 3. Attention Sink 与 StreamingLLM

简单删除最早 token 会破坏注意力分布。StreamingLLM 发现开头少量 token 常充当 attention sink，因此保留：

```text
[最初 S 个 sink token] + [最近 W 个 token]
```

可稳定流式处理更长序列。它适合连续生成，但不等价于完整长上下文理解：被删除的中间事实仍无法访问。

## 4. KV Cache 量化与压缩

- FP16→FP8/INT8：容量和带宽约减半；
- INT4：进一步压缩但精度与反量化更难；
- token eviction：按重要性删除 KV；
- merging/compression：把多个历史 token 压成摘要表示。

压缩方法必须分别评测短任务精度和 needle-in-a-haystack、长文档 QA 等长上下文能力。

## 5. KV Offload

把冷 KV 放到 CPU、NVMe 或远端 KV store。设要恢复的数据为 $`S`$：

```math
T_{reload}\ge S/BW_{link}.
```

Offload 解决容量，不自动解决延迟。需要预取、冷热分层和与计算重叠。

## 6. Chunked Prefill

把长 prompt 拆分，主要改善调度与尾延迟；总 attention 计算并不会凭空减少。若命中 prefix cache，才真正跳过已缓存前缀的 prefill。

## 7. Context Parallelism

将序列/KV 分到多卡。Ring Attention 让 query block 在设备间与 K/V block 交互，并将通信与计算重叠。它降低单卡容量压力，但要求高速互联。

## 8. 稀疏与分块注意力

- local + global token；
- block sparse；
- retrieval/selective attention；
- recurrent memory。

它们能改变理论复杂度，但通常需要模型结构或训练方式配合，不能作为任意 checkpoint 的无损替换。

## 9. Prefix Cache 与 RAG

对重复长文档，prefix cache 可直接复用 KV；但动态 RAG 文档顺序、模板或时间戳变化都会破坏前缀匹配。为了命中缓存，可以：

- 固定 system prompt 和文档排序；
- 把高复用内容放在前缀；
- 避免在前缀中插入随机字段；
- 对租户与模型版本做缓存隔离。

## 10. 长上下文并不等于有效上下文

模型支持 1M token 只说明位置编码和运行时允许该长度，不代表所有位置都能可靠检索和推理。必须同时测：

- passkey/needle retrieval；
- 多 needle；
- 长文档聚合推理；
- 位置偏置；
- 长输出稳定性；
- TTFT、TPOT 与成本。

## 11. 面试回答

> 长上下文有四重成本：Prefill dense attention 近似平方增长，KV 容量线性增长，Decode 每步读取 KV 线性增长，长 Prefill 还会阻塞调度。FlashAttention、chunking、KV 量化/Offload、窗口注意力和 Context Parallel 分别解决不同瓶颈。Sliding window 或 KV eviction 会丢信息，不能宣称与完整 attention 等价。
